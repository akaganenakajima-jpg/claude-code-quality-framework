# Claude Code 品質フレームワーク

**プロンプト1つで Claude Code を「体系的に考え、専門知識に基づいて行動するプロフェッショナル」に変えるパッケージ。**

タスクを受けると自分で課題を分析し、8分野の専門知識から必要な情報を引き出し、品質規律に従って作業する — そんな認知フレームワークを Claude Code に導入します。

## 3本柱

```
 判断                    知識                      規律
 ─────                  ─────                    ─────
 タスクを受ける          8分野・58ファイルの        PDCA・リスク判定・
    ↓                   専門知識を                 5Sで毎回同じ
 13大分類+8横断で        オンデマンドで             品質を出す
 課題を自動分析          引き出す
    ↓                      ↓                        ↓
 CLAUDE.md              knowledge/                quality/
 (~160行・毎回読込)      (必要な§だけ読む)          (省略禁止のゲート)
```

| 柱 | 役割 | 仕組み |
|---|---|---|
| **判断** | 何を参照すべきか自分で決める | CLAUDE.md が13大分類+8横断チェックでタスクを分析→知識ファイルへ自動ルーティング |
| **知識** | 何を知っているか | 8分野58ファイルの専門知識をオンデマンド供給（平均1-2ファイル/タスク） |
| **規律** | どう動くか | ISO 9001準拠のPDCA・リスク判定・5Sで全タスクの品質を統制 |

## セットアップ

### Step 1: プロンプトをコピー
`setup-prompt.md` を開き、**内容を全てコピー**してください。

### Step 2: Claude Code に貼り付け
Claude Code を起動し、コピーした内容をチャット欄に貼り付けて送信してください。

### Step 3: 指示に従って進める
Claude Code が自動的に全ファイルを作成します。
ファイル作成の確認が出たら許可してください。

**以上で完了です。**

### アップデート
既にインストール済みの場合、同じ手順（Step 1〜3）を再実行するだけでOKです。
Claude が自動的に既存ファイルとの差分を検出し、変更があるファイルだけを更新します。

## どう動くか

### 自動ルーティングの例

「Webアプリの認証機能を実装して」と言うと:

1. **CLAUDE.md が課題を分析** → 大分類: 🏗️ つくる / 横断: 🛡️ 安全
2. **知識ファイルを自動Read** → `ipa/ap.md`（実装パターン）+ `ipa/au.md`（認証認可）
3. **品質規律に従って作業** → Plan（受入基準）→ Do（TDD）→ Check（3層検証）→ Act

### スラッシュコマンド

| コマンド | 用途 |
|---------|------|
| `/5s` | 作業開始前の準備チェック（片付け・現状確認・リソース確認） |
| `/knowledge` | 課題分析→知識ベース自動参照 |
| `/quality-review` | PDCAゲート完了状況の確認 |
| `/risk` | git diff から変更リスクを自動判定（🟢🟡🟠🔴） |

### Git Hook（自動リマインダー）

コミット時に変更量を分析し、リスクレベルを自動表示:
```
⚙️ リスク判定: 🟡 中リスク（5ファイル / +120 -30 行）
→ 品質ゲートを確認してください
```

## 設計思想

### オンデマンド読み込み

```
CLAUDE.md（~160行）          ← 毎セッション読み込み。判定・トリガーのみ
  ├─ 大分類 13項目           「何をする課題か？」→ 参照先を特定
  └─ 横断チェック 8項目      「安全？性能？テスト？UX？」→ 追加参照
       ↓
knowledge/*/index.md          ← オンデマンド読み込み。シナリオ逆引きでルーティング
       ↓
knowledge/*/detail.md          ← 必要な§セクションだけ読む（平均1-2ファイル）
```

CLAUDE.md は約160行のコンパクトな判定エンジン。58ファイルの知識を毎回読み込むのではなく、必要なときに必要な部分だけ参照することで、コンテキスト消費を最小化しています。

### ECC との共存

[everything-claude-code](https://github.com/affaan-m/everything-claude-code) (ECC) と**競合ゼロ**で併用できます。

| 領域 | 当フレームワーク | ECC |
|---|---|---|
| 対象 | **プロセス品質**（判断/知識/規律） | **コード品質**（フォーマット/型/テスト実行） |
| 格納先 | `~/.claude/CLAUDE.md` + `knowledge/` | `~/.claude/rules/` |
| Hook | リスク判定リマインダー | lint/type check/test |
| コマンド | `/5s`, `/knowledge`, `/risk`, `/quality-review` | ECC固有の57コマンド |

## カスタマイズ

### プロジェクト固有ルールの追加
各プロジェクトのリポジトリに `CLAUDE.md` を作成し、プロジェクト固有のルールを記載してください。
テンプレートは `examples/project-claude-md.example` を参照してください。

### Hookのカスタマイズ
`hooks/process-gate.py` をベースに、プロジェクト固有のリスク判定ロジックを追加できます。
`~/.claude/settings.json` の `hooks` セクションで設定します。
