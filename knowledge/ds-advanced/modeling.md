### 1.2 学習/検証/テスト分割戦略

| 手法 | 説明 | 適用場面 |
|---|---|---|
| ホールドアウト | 学習:検証:テスト = 6:2:2 | 大規模データ（10万行以上） |
| k-Fold CV | データをk分割、k-1で学習・1で評価を繰り返す | 中規模データ |
| Stratified k-Fold | クラス比率を保持したk-Fold | 不均衡データの分類 |
| Group k-Fold | グループ（ユーザー等）単位で分割 | データ漏洩防止 |
| Time Series Split | 時系列の因果関係を保持 | 時系列データ（未来→過去の漏洩防止） |
| Nested CV | 外側CV（評価）×内側CV（チューニング） | 汎化性能の不偏推定 |
| Leave-One-Out CV | n分割（1サンプルずつ検証） | 極小データセット |

**データ漏洩 (Leakage) の主な原因**:
- テストデータの情報が学習に混入（前処理の全データfit、未来の情報使用）
- ターゲットから直接導出できる特徴量の使用
- グループ情報を無視した分割（同一患者の複数レコード等）

### 1.3 特徴量選択

| 手法 | 原理 | 特徴 |
|---|---|---|
| **フィルタ法** | 統計量で個別評価 | 高速、モデル非依存 |
| - 分散フィルタ | 分散が閾値以下を除外 | 定数に近い特徴量を除去 |
| - 相関フィルタ | 目的変数との相関で選択 | 線形関係のみ検出 |
| - 相互情報量 | 非線形依存性も検出 | 計算コストやや高い |
| - カイ二乗検定 | カテゴリ変数と目的変数の独立性 | カテゴリカル特徴量向け |
| **ラッパー法** | モデル性能で評価 | 高精度、計算コスト高 |
| - 前進選択法 | 1つずつ追加 | 変数が少ない場合 |
| - 後退除去法 | 1つずつ除外 | 変数が少ない場合 |
| - RFE (Recursive Feature Elimination) | 重要度最低を逐次除外 | 中規模 |
| **埋め込み法** | 学習と同時に選択 | モデル固有 |
| - LASSO (L1正則化) | 係数を0に縮退 | 線形モデル |
| - 木モデルの重要度 | 不純度減少 / 順列重要度 | 非線形対応 |
| - ElasticNet | L1+L2 | 相関する特徴量群 |

### 1.4 ハイパーパラメータチューニング

| 手法 | 探索方式 | 効率 | 特徴 |
|---|---|---|---|
| グリッドサーチ | 全組み合わせ | 低 | 確実だがスケールしない |
| ランダムサーチ | ランダムサンプリング | 中 | 高次元で効率的（Bergstra & Bengio 2012） |
| ベイズ最適化 | 獲得関数で次の候補を選択 | 高 | Optuna, scikit-optimize |
| Hyperband | Early Stopping + ランダム | 高 | 計算資源を効率配分 |
| BOHB | ベイズ + Hyperband | 最高 | Optuna integrationあり |

**Optuna の基本パターン**:
```python
import optuna

def objective(trial):
    params = {
        'n_estimators': trial.suggest_int('n_estimators', 100, 1000),
        'max_depth': trial.suggest_int('max_depth', 3, 10),
        'learning_rate': trial.suggest_float('learning_rate', 1e-3, 0.3, log=True),
        'subsample': trial.suggest_float('subsample', 0.5, 1.0),
    }
    model = LGBMClassifier(**params)
    score = cross_val_score(model, X, y, cv=5, scoring='roc_auc')
    return score.mean()

study = optuna.create_study(direction='maximize')
study.optimize(objective, n_trials=100)
```

## 2. 回帰モデル

### 2.1 線形回帰・正則化回帰

**線形回帰**: y = Xβ + ε
- 最小二乗法 (OLS): Σ(yᵢ - ŷᵢ)² を最小化
- 仮定: 線形性、独立性、等分散性、正規性、多重共線性なし
- VIF (Variance Inflation Factor): VIF > 10 → 多重共線性の疑い

**正則化回帰**:

| 手法 | ペナルティ項 | 特徴 |
|---|---|---|
| Ridge (L2) | λΣβⱼ² | 係数を縮小、全変数残る |
| LASSO (L1) | λΣ\|βⱼ\| | 一部の係数を0に（変数選択） |
| ElasticNet | αλΣ\|βⱼ\| + (1-α)λΣβⱼ² | Ridge+LASSOの組み合わせ |

**正則化パラメータλの選択**: クロスバリデーションで最適λを探索

### 2.2 決定木・アンサンブル回帰

**決定木回帰**:
- 分割基準: MSE（平均二乗誤差）の減少量
- 過学習しやすい → 剪定（max_depth, min_samples_leaf等）

**ランダムフォレスト回帰**:
- バギング + 特徴量のランダムサンプリング
- Out-of-Bag (OOB) エラーでバリデーション不要の推定
- 過学習に強い、並列化可能

**勾配ブースティング回帰**:

| パラメータ | XGBoost | LightGBM | CatBoost |
|---|---|---|---|
| 分割方式 | レベルワイズ | リーフワイズ | 対称木 |
| カテゴリ変数 | 手動エンコーディング | 直接対応 | 直接対応（最適） |
| 欠損値 | 自動処理 | 自動処理 | 自動処理 |
| 速度 | 中 | 高速 | 中 |
| 精度 | 高 | 高 | 高（小データで優位） |

**主要ハイパーパラメータ（GBDT共通）**:
- `n_estimators`: 木の本数（多い→過学習、early_stoppingで制御）
- `learning_rate`: 学習率（小さい→多くの木が必要、汎化性能向上）
- `max_depth`: 木の深さ（深い→複雑なパターン、過学習リスク）
- `subsample`: データのサンプリング率（過学習防止）
- `colsample_bytree`: 特徴量のサンプリング率（多重共線性緩和）
- `min_child_weight` / `min_data_in_leaf`: リーフの最小サンプル数（過学習防止）
- `reg_alpha` (L1) / `reg_lambda` (L2): 正則化

### 2.3 評価指標（回帰）

| 指標 | 式 | 特徴 |
|---|---|---|
| RMSE | √(Σ(yᵢ-ŷᵢ)²/n) | 大きな誤差に敏感。元の単位 |
| MAE | Σ\|yᵢ-ŷᵢ\|/n | 外れ値にロバスト。元の単位 |
| MAPE | Σ\|yᵢ-ŷᵢ\|/yᵢ/n × 100% | スケール非依存。yᵢ≈0で不安定 |
| R² | 1 - SS_res/SS_tot | 分散の説明割合。1に近いほど良い |
| Adjusted R² | 1 - (1-R²)(n-1)/(n-p-1) | 変数数でペナルティ。変数選択に使用 |
| RMSLE | √(Σ(log(ŷᵢ+1)-log(yᵢ+1))²/n) | 相対誤差を重視、右裾重い分布向け |

**指標選択の指針**:
- ビジネスで大きな外れが致命的 → RMSE
- 外れ値の影響を抑えたい → MAE
- 相対的な精度が重要 → MAPE
- 値のスケールが大きく変動 → RMSLE

### 2.4 実務活用: 回帰モデル構築

## 3. 分類モデル

### 3.1 ロジスティック回帰

**モデル**: P(y=1|x) = σ(Xβ) = 1 / (1 + exp(-Xβ))
- 線形決定境界。解釈性が高い（係数の指数 = オッズ比）
- 正則化: L1（スパース）、L2（縮小）、ElasticNet
- 多クラス: softmax回帰（multinomial）、One-vs-Rest

### 3.2 SVM (Support Vector Machine)

- **線形SVM**: マージン最大化。サポートベクターのみが決定境界を決定
- **カーネルSVM**: カーネルトリックで非線形境界（RBF, 多項式, sigmoid）
- **パラメータ**: C（正則化。小→マージン広、誤分類許容）、γ（RBFカーネルの幅）
- 大規模データには不向き（O(n²)〜O(n³)）

### 3.3 アンサンブル手法

| 手法 | 原理 | 分散削減 | バイアス削減 |
|---|---|---|---|
| バギング (RF) | ブートストラップ + 並列学習 + 平均/多数決 | 強い | 弱い |
| ブースティング (GBDT) | 残差に逐次フィット | 弱い | 強い |
| スタッキング | メタ学習器で組み合わせ | 中 | 中 |
| ブレンディング | 重み付き平均 | 中 | 弱い |

**バイアス-バリアンストレードオフ**:
- 高バイアス（未学習）: モデルが単純すぎる → 複雑なモデル / 特徴量追加
- 高バリアンス（過学習）: モデルが複雑すぎる → 正則化 / データ増加 / アンサンブル
- 学習曲線: 学習データサイズ vs 学習/検証スコアのプロットで診断

### 3.4 評価指標（分類）

| 指標 | 式 | 用途 |
|---|---|---|
| Accuracy | (TP+TN)/(TP+TN+FP+FN) | 均衡データの全体的な正確さ |
| Precision | TP/(TP+FP) | 偽陽性を減らしたい場合（スパム検知） |
| Recall | TP/(TP+FN) | 偽陰性を減らしたい場合（疾病検出） |
| F1 | 2×P×R/(P+R) | Precision/Recallのバランス |
| Fβ | (1+β²)×P×R/(β²×P+R) | β>1:Recall重視、β<1:Precision重視 |
| AUC-ROC | ROC曲線の下面積 | 閾値非依存の識別能力 |
| AUC-PR | PR曲線の下面積 | 不均衡データでROCより適切 |
| Log Loss | -Σ(y log ŷ + (1-y)log(1-ŷ))/n | 確率推定の精度 |
| Cohen's Kappa | (accuracy - chance) / (1 - chance) | 偶然一致を補正した精度 |

**混同行列**:
```
              予測: Positive  予測: Negative
実際: Positive    TP              FN
実際: Negative    FP              TN
```

**ROC曲線**: TPR(Recall) vs FPR(1-Specificity) の閾値変化プロット
- AUC = 0.5 → ランダム、AUC = 1.0 → 完全分類
- 不均衡データでは楽観的に見えるため PR曲線を併用

### 3.5 不均衡データ対策

| 手法 | カテゴリ | 説明 |
|---|---|---|
| ランダムオーバーサンプリング | リサンプリング | 少数クラスを複製（過学習リスク） |
| SMOTE | リサンプリング | 少数クラスを合成（k-NN補間） |
| ADASYN | リサンプリング | 学習困難な領域を重点的に合成 |
| ランダムアンダーサンプリング | リサンプリング | 多数クラスを削減（情報損失） |
| Tomek Links | クリーニング | 境界付近のノイズ除去 |
| コスト敏感学習 | アルゴリズム | class_weightで誤分類コストを調整 |
| 閾値調整 | 後処理 | 確率閾値を0.5以外に設定 |
| Focal Loss | 損失関数 | 簡単なサンプルの損失を抑制 |
| アンサンブル | アルゴリズム | BalancedRF, EasyEnsemble |

**SMOTEの原理**:
1. 少数クラスのサンプルxを選択
2. xのk近傍（k-NN）からランダムに1つ（x'）を選択
3. x と x' の間のランダムな点を新サンプルとして生成

**注意点**:
- SMOTE/オーバーサンプリングは学習データにのみ適用（検証/テストには適用しない）
- Pipeline内で `imblearn.pipeline.Pipeline` を使用してリーク防止

### 3.6 実務活用: 分類モデル構築

**不均衡データでの推奨フロー**:
1. まず `class_weight='balanced'` で試行
2. 閾値調整: PR曲線からF1最大の閾値を選択
3. それでも不足 → SMOTE + Tomek Links
4. 最終手段 → Focal Loss / アンサンブル

## 4. 教師なし学習

### 4.1 クラスタリング

| 手法 | 原理 | クラスタ形状 | パラメータ |
|---|---|---|---|
| k-means | 重心最小化 | 球状 | k（クラスタ数） |
| k-means++ | 初期値改善版k-means | 球状 | k |
| Mini-Batch k-means | サンプリングで高速化 | 球状 | k, batch_size |
| DBSCAN | 密度ベース | 任意形状 | eps, min_samples |
| HDBSCAN | 階層的DBSCAN | 任意形状 | min_cluster_size |
| 階層的 (Agglomerative) | ボトムアップ結合 | デンドログラム | 距離基準, カット高さ |
| GMM | ガウス混合モデル | 楕円体 | n_components |
| Spectral | グラフラプラシアン | 任意形状 | k, affinity |

**クラスタ数の決定**:
- **エルボー法**: SSE（クラスタ内二乗和）の減少が緩やかになる点
- **シルエット分析**: シルエットスコアの最大化（-1〜1、高いほど良い）
- **Gap統計量**: 実データとランダムデータのWSS差
- **BIC/AIC**: GMMの場合の情報量基準

**シルエットスコア**:
```
s(i) = (b(i) - a(i)) / max(a(i), b(i))
a(i): 同クラスタ内の平均距離
b(i): 最近隣クラスタとの平均距離
```

### 4.2 次元削減

| 手法 | 原理 | 線形/非線形 | 用途 |
|---|---|---|---|
| PCA | 分散最大化方向に射影 | 線形 | 次元削減、ノイズ除去 |
| Kernel PCA | カーネルトリック+PCA | 非線形 | 非線形構造の抽出 |
| t-SNE | 確率分布の近傍保存 | 非線形 | 2D/3D可視化（大域構造弱い） |
| UMAP | 位相構造の保存 | 非線形 | 可視化+クラスタリング前処理 |
| LDA (Linear Discriminant) | クラス分離最大化 | 線形 | 教師あり次元削減 |
| NMF | 非負制約付き行列分解 | 線形 | テキスト、画像の部品抽出 |
| Autoencoder | NN圧縮+復元 | 非線形 | 深層学習の表現学習 |

**PCAの手順**:
1. データを標準化（平均0、分散1）
2. 共分散行列を計算
3. 固有値分解 → 固有ベクトル（主成分方向）と固有値（分散寄与）
4. 累積寄与率が閾値（例: 95%）に達する主成分数を選択
5. 選択した主成分に射影

**t-SNE vs UMAP**:
- t-SNE: 局所構造の保存に優れるが大域構造が歪む、計算コスト O(n²)
- UMAP: 大域+局所構造を保存、高速、クラスタリングの前処理にも使える

### 4.3 異常検知

| 手法 | 原理 | 特徴 |
|---|---|---|
| Isolation Forest | ランダム分割の孤立深さ | 高次元、高速、アンサンブル |
| LOF (Local Outlier Factor) | 局所密度比 | 密度不均一に強い |
| One-Class SVM | 正常データの境界学習 | カーネルで非線形対応 |
| Elliptic Envelope | ロバスト共分散推定 | 正規分布仮定 |
| Autoencoder | 復元誤差 | 深層学習、高次元 |
| k-NN距離 | k近傍までの距離 | シンプル、解釈容易 |

**Isolation Forestの原理**:
- ランダムな特徴量＋ランダムな閾値で二分木を構築
- 異常値は少ない分割で孤立する → 平均パス長が短い
- アノマリースコア: s(x) = 2^(-E(h(x))/c(n))（正規化パス長）

### 4.4 実務活用: 顧客セグメンテーション

**RFM分析 + クラスタリング**:
1. RFM特徴量: Recency（最終購入日からの日数）、Frequency（購入頻度）、Monetary（購入金額）
2. 対数変換（右裾の重い分布を正規化）

5. 各セグメントのプロファイリング（RFMスコア、行動パターン）
6. セグメント別マーケティング施策の設計

## 5. モデル解釈性

### 5.1 SHAP (SHapley Additive exPlanations)

**理論**: 協力ゲーム理論のShapley値を特徴量の貢献度に応用
- 各特徴量の「限界貢献度」の加重平均
- 公理的に一意（効率性、対称性、ダミー、加法性）

**SHAP値の種類**:
- **TreeSHAP**: 木モデル専用の高速アルゴリズム O(TLD²)
- **KernelSHAP**: モデル非依存（任意のブラックボックスモデル）
- **DeepSHAP**: 深層学習向け（DeepLIFT + SHAP）

**可視化**:
- **Summary Plot**: 全特徴量のSHAP値の分布（重要度+効果方向）
- **Dependence Plot**: 特定の特徴量値 vs SHAP値（非線形効果の確認）
- **Force Plot**: 個別予測の説明（各特徴量がベースラインからどう押したか）
- **Waterfall Plot**: 個別予測の特徴量貢献の積み上げ

```python
import shap

explainer = shap.TreeExplainer(model)
shap_values = explainer(X_test)

# Summary Plot（グローバルな特徴量重要度）
shap.summary_plot(shap_values, X_test)

# 個別予測の説明
shap.waterfall_plot(shap_values[0])
```

### 5.2 LIME (Local Interpretable Model-agnostic Explanations)

**原理**: 予測対象の周辺でデータを摂動 → 局所的に線形モデルで近似

**手順**:
1. 説明したいインスタンス x の周辺にサンプルを生成
2. 各サンプルのブラックボックス予測を取得
3. x からの距離で重み付け
4. 重み付き線形回帰でフィット → 係数が特徴量の局所的重要度

**SHAP vs LIME**:

| 観点 | SHAP | LIME |
|---|---|---|
| 理論的基盤 | Shapley値（公理的） | 局所線形近似 |
| 一貫性 | 保証あり | サンプリング依存で不安定 |
| 計算コスト | TreeSHAPは高速 | ブラックボックスは遅い |
| グローバル説明 | Summary Plotで対応 | 個別説明の集約が必要 |

### 5.3 Partial Dependence Plot (PDP) / ICE

**PDP**: 特定の特徴量と予測値の周辺効果（他の特徴量を平均化）
- 特徴量間に相関がある場合、非現実的な組み合わせを含む可能性

**ICE (Individual Conditional Expectations)**: 各インスタンスの依存関係を個別にプロット
- PDPは平均→ ICEは個別のばらつきが見える
- 交互作用効果の発見に有用

**Accumulated Local Effects (ALE)**:
- PDPの問題（特徴量相関）を解決
- 局所的な効果を累積。相関する特徴量がある場合により正確

### 5.4 変数重要度

| 手法 | 原理 | 注意点 |
|---|---|---|
| 不純度重要度 (MDI) | 分割での不純度減少量の合計 | 高カーディナリティにバイアス |
| 順列重要度 (MPI) | 特徴量をシャッフルした時の性能低下 | 相関する特徴量で過小評価 |
| Drop-Column重要度 | 特徴量を除外して再学習 | 計算コスト高いが最も正確 |
| SHAP重要度 | SHAP値の絶対値平均 | 理論的に最も堅牢 |
| Boruta | 影変数との比較で有意な変数を選択 | 統計的に厳密 |

### 5.5 実務活用: モデルの説明責任

**説明のレベル**:
1. **グローバル**: モデル全体の重要特徴量（SHAP Summary、変数重要度）
2. **コホート**: セグメント別の特徴量効果（PDP/ALE）
3. **個別**: 個々の予測の根拠（SHAP Waterfall、LIME）

**金融規制（バーゼル等）での解釈性要件**:
- 融資判断: 拒否理由の説明義務 → SHAP個別説明
- リスクモデル: 規制当局への説明 → PDP + 変数重要度
- 「モデルの精度」と「解釈性」のトレードオフを明示的に管理

## 6. MLOps基礎

### 6.1 MLパイプライン

**コンポーネント**:
1. **データ取得**: スケジュール or イベントトリガー
2. **データ検証**: スキーマ・品質チェック
3. **前処理**: 特徴量エンジニアリング
4. **学習**: モデルトレーニング + バリデーション
5. **評価**: テストデータでの性能評価
6. **登録**: モデルレジストリへの登録
7. **デプロイ**: サービング環境へのリリース
8. **監視**: 性能・データドリフトの継続監視

### 6.2 モデルのバージョン管理

**MLflow**:
- **Tracking**: 実験のパラメータ・メトリクス・アーティファクトを記録
- **Models**: モデルのパッケージング（多フレームワーク対応）
- **Registry**: モデルのステージ管理（Staging → Production → Archived）
- **Projects**: 再現性のある実行環境定義

**モデルレジストリの必須メタデータ**:
- モデルバージョン、学習日時、学習データのバージョン
- 評価メトリクス（テストデータでの性能）
- 特徴量リスト、ハイパーパラメータ
- 承認者、デプロイ環境

### 6.3 モデル監視

**データドリフト**: 入力データの分布が変化
- 検出: KL-divergence、KS検定、PSI (Population Stability Index)
- PSI < 0.1: 安定、0.1〜0.25: 注意、> 0.25: 要再学習

**概念ドリフト**: 入力→出力の関係が変化（同じ特徴量でも結果が異なる）
- 検出: 予測性能のモニタリング（AUC, RMSE等の時系列推移）
- ADWIN (Adaptive Windowing): ストリームデータでのドリフト検出

**ラベルドリフト**: 目的変数の分布が変化

**再学習戦略**:
- **定期再学習**: 週次/月次でスケジュール実行
- **トリガーベース**: ドリフト検出時に自動再学習
- **オンライン学習**: データ到着ごとにインクリメンタル更新

### 6.4 モデルサービング

| パターン | 特徴 | レイテンシ | 用途 |
|---|---|---|---|
| バッチ推論 | 定期実行、結果をDB/KVに保存 | 高（許容） | レコメンド、スコアリング |
| リアルタイムAPI | REST/gRPC | 低（ms〜s） | 検索ランキング、不正検知 |
| ストリーミング | イベント駆動 | 低〜中 | リアルタイム異常検知 |
| エッジ推論 | デバイス上で推論 | 極低 | IoT、モバイル |

**Shadow Deployment**: 新モデルを本番トラフィックで並行実行、結果のみ記録（影響なし）
**Canary Deployment**: 少数のトラフィック（1〜5%）を新モデルに振り向け、問題なければ拡大

### 6.5 実務活用: MLOps成熟度モデル

| レベル | 特徴 | 自動化 |
|---|---|---|
| 0 | 手動（Jupyter Notebook → 手動デプロイ） | なし |
| 1 | ML Pipeline自動化（学習〜評価の自動実行） | 学習パイプライン |
| 2 | CI/CD統合（モデル変更→自動テスト→自動デプロイ） | 学習+デプロイ |
| 3 | CT (Continuous Training)（ドリフト検出→自動再学習→自動デプロイ） | 全自動 |

**最低限のMLOps（レベル1を目指す）**:
1. 実験管理: MLflow or Weights & Biases
2. コード管理: Git（notebooks含む）
3. データバージョニング: DVC or Delta Lake
4. モデルレジストリ: MLflow Model Registry
5. 監視: 予測分布の日次チェック
