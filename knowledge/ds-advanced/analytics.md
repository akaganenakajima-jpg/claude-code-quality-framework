## 1. 探索的データ分析 (EDA)

### 1.1 記述統計

#### 位置の要約（中心傾向）

| 指標 | 式 | 特徴 |
|---|---|---|
| 算術平均 | Σxᵢ/n | 外れ値に敏感 |
| 中央値 | ソート後の中央 | 外れ値にロバスト |
| 最頻値 | 最も頻度が高い値 | カテゴリカルにも適用可 |
| トリム平均 | 上下α%を除いた平均 | 外れ値を部分的に除外 |
| 加重平均 | Σwᵢxᵢ/Σwᵢ | サンプル重要度が異なる場合 |

#### 散布度（ばらつき）

| 指標 | 式 | 特徴 |
|---|---|---|
| 分散 | Σ(xᵢ-x̄)²/(n-1) | 不偏推定量（n-1で割る） |
| 標準偏差 | √分散 | 元の単位と同じ |
| 変動係数 (CV) | σ/μ | 単位に依存しない相対的ばらつき |
| IQR | Q3 - Q1 | ロバスト、箱ひげ図の箱の幅 |
| MAD | median(\|xᵢ - median(x)\|) | ロバストな散布度 |
| レンジ | max - min | 最も単純、外れ値に敏感 |

#### 形状の要約

| 指標 | 値の解釈 |
|---|---|
| 歪度 (Skewness) | = 0: 対称, > 0: 右裾長い（正の歪み）, < 0: 左裾長い |
| 尖度 (Kurtosis) | = 3: 正規分布（超過尖度0）, > 3: 裾が重い, < 3: 裾が軽い |

**正規性の検定**:
- Shapiro-Wilk検定: 小〜中標本（n < 5000）で検出力が高い
- Kolmogorov-Smirnov検定: 分布の適合度検定（一般化）
- Anderson-Darling検定: 裾の適合度に重み付け
- Q-Qプロット: 視覚的判断（対角線からの乖離）

### 1.2 相関分析

| 指標 | 前提 | 範囲 | 検出できる関係 |
|---|---|---|---|
| ピアソン相関係数 (r) | 連続変数、線形関係 | [-1, 1] | 線形相関のみ |
| スピアマン順位相関 (ρ) | 順序尺度以上 | [-1, 1] | 単調関係（非線形含む） |
| ケンドール順位相関 (τ) | 順序尺度以上 | [-1, 1] | 順位の一致度（小標本で安定） |
| 点双列相関 | 一方が2値、他方が連続 | [-1, 1] | 2群間の差 |
| クラメールのV | 名義尺度×名義尺度 | [0, 1] | カテゴリ間の関連 |
| 相互情報量 | 任意 | [0, ∞) | 非線形含む任意の依存性 |

**相関と因果の違い**:
- 相関 ≠ 因果。擬似相関（交絡変数）の可能性を常に考慮
- 因果推論手法: RCT（ランダム化比較試験）、回帰不連続デザイン、差分の差分法、傾向スコアマッチング、操作変数法

**相関行列の活用**:
- 多重共線性（VIF > 10）の検出 → 変数除外 or 正則化
- ヒートマップで視覚化 → 特徴量グルーピングの手がかり

### 1.3 クロス集計・ピボットテーブル

**分析パターン**:
- 2変数の関連性: カイ二乗検定、Fisherの正確検定（小標本）
- 3変数以上: 層化分析（シンプソンのパラドックス回避）
- 集約関数: count/sum/mean/median/std

**シンプソンのパラドックス**: 全体と部分で傾向が逆転する現象
- 例: 全体では治療群の生存率が高いが、重症度で層別すると非治療群が優位
- 対策: 交絡変数で層化 or 多変量モデル

### 1.4 実務活用: EDAワークフロー

```python
# 1. データ概要
df.info()        # 型、非NULL数
df.describe()    # 基本統計量
df.shape         # 行×列

# 2. 欠損値パターン
import missingno as msno

# 3. 分布確認

# 4. 相関

# 5. 自動プロファイリング
from ydata_profiling import ProfileReport

```

## 2. 可視化の原則

### 2.1 タフテの原則

**データインク比 (Data-Ink Ratio)**:
- データインク比 = データを表すインクの量 / グラフ全体のインクの量
- 最大化すべき: 装飾的要素（3Dエフェクト、グリッド線過多、影）を排除
- "Above all else, show the data" — Edward Tufte

**その他のタフテ原則**:
- **チャートジャンク回避**: 不要な装飾はノイズ
- **スモールマルチプル**: 同じスケールの小グラフを並べて比較
- **データ密度**: 限られたスペースに多くの情報を
- **Lie Factor**: 視覚的効果/データの変化 = 1 に近づける（歪曲回避）

### 2.2 グラフ選択マトリクス

| 目的 | 推奨グラフ | 避けるべきグラフ |
|---|---|---|
| **比較**（カテゴリ間） | 棒グラフ、ドットプロット | 円グラフ（比較困難） |
| **推移**（時系列） | 折れ線グラフ、面グラフ | 棒グラフ（連続性が伝わらない） |
| **構成**（割合） | 積み上げ棒、ツリーマップ | 3D円グラフ（角度の歪み） |
| **分布** | ヒストグラム、箱ひげ図、バイオリンプロット | 棒グラフ（分布が見えない） |
| **関係** | 散布図、バブルチャート | 折れ線グラフ（因果を暗示） |
| **地理** | コロプレスマップ、バブルマップ | 3Dマップ（読み取り困難） |
| **フロー** | サンキーダイアグラム、アルビアルダイアグラム | 円グラフ |
| **階層** | ツリーマップ、サンバーストチャート | 棒グラフ（階層が表現できない） |

**グラフ選択の判断フロー**:
1. 何を伝えたいか（比較/推移/構成/分布/関係）を明確に
2. データ型（連続/離散/カテゴリ/時系列）を確認
3. 聴衆のリテラシーを考慮（一般向け → シンプルなグラフ）
4. インタラクティブ性の要否を判断

### 2.3 色覚バリアフリー

**カラーユニバーサルデザイン (CUD)**:
- 色覚多様性: 日本男性の約5%、女性の約0.2%が色覚異常
- 赤緑の区別が困難な人が最も多い（P型/D型）

**対策**:
- 赤と緑の同時使用を避ける → 青とオレンジ、青と赤
- 色だけでなく形状（マーカー形状、線種）で区別
- 色覚シミュレーター（Coblis等）で確認
- 推奨カラーパレット: `viridis`, `cividis`, `colorblind`（seaborn）

### 2.4 ダッシュボード設計原則

**レイアウト**:
- **逆ピラミッド型**: KPIサマリ → トレンド → 詳細（上から下へ詳細度増加）
- **Zパターン**: 左上→右上→左下→右下の視線移動に沿う
- **5秒ルール**: 5秒以内に最重要情報が読み取れること

**設計チェックリスト**:
1. 対象ユーザーと意思決定シーンを明確化
2. KPIは3〜5個に絞る（情報過多は逆効果）
3. フィルタ（日付、セグメント）を提供
4. ドリルダウン可能な構造
5. 更新頻度をデータの性質に合わせる（リアルタイム or 日次）
6. モバイル対応（レスポンシブ）

### 2.5 実務活用: 可視化ツールの使い分け

| ツール | 特徴 | 適用場面 |
|---|---|---|
| matplotlib | 低レベル、カスタマイズ自在 | 論文・レポート用の静的グラフ |
| seaborn | 統計グラフ特化、美しいデフォルト | EDA、統計分析 |
| plotly | インタラクティブ、Web対応 | ダッシュボード、プレゼン |
| Altair | 宣言的文法、Vega-Lite | 探索的分析、再現性重視 |
| Tableau/Looker | BI特化、非エンジニア向け | 社内ダッシュボード |
| Streamlit | Python→Webアプリ | プロトタイプ、データアプリ |

## 3. A/Bテスト設計

### 3.1 仮説設定

**基本構造**:
- **帰無仮説 (H₀)**: 「AとBに差はない」（現状維持）
- **対立仮説 (H₁)**: 「AとBに差がある」（改善効果あり）
- 片側検定 vs 両側検定: 改善方向が明確なら片側、不明なら両側

**指標の選定**:
- **主要指標 (Primary Metric)**: ビジネス目標に直結（CVR、ARPU等）。1つに絞る
- **副次指標 (Secondary Metric)**: 副作用の監視（離脱率、ページ滞在時間等）
- **ガードレール指標**: 悪化してはならない指標（エラー率、レイテンシ等）
- **OEC (Overall Evaluation Criterion)**: 複数指標の加重合成（高度）

### 3.2 サンプルサイズ計算

**必要パラメータ**:
- **有意水準 (α)**: 第1種の過誤（偽陽性）の許容確率。通常 0.05
- **検出力 (1-β)**: 第2種の過誤（偽陰性）を避ける確率。通常 0.80
- **最小検出効果量 (MDE)**: 検出したい最小の差。ビジネス要件から決定
- **ベースラインの分散**: 過去データから推定

```
# 2群の比率の差のサンプルサイズ（近似式）
n = (Z_{α/2} + Z_β)² × (p₁(1-p₁) + p₂(1-p₂)) / (p₁ - p₂)²

# 例: CVR 5%→6% (MDE=1%), α=0.05, β=0.20
# n ≈ 4,900 / 群
```

**検出力分析の考え方**:
- α↓（厳しく）→ 必要サンプルサイズ↑
- MDE↓（小さい差を検出）→ 必要サンプルサイズ↑↑
- 検出力↑ → 必要サンプルサイズ↑

### 3.3 ランダム化手法

| 手法 | 説明 | 注意点 |
|---|---|---|
| ユーザー単位 | ユーザーIDでランダム割り当て | 最も一般的。一貫性あり |
| セッション単位 | セッションごとにランダム | 同一ユーザーが両群に入る可能性 |
| ページ単位 | ページビューごとに | 体験の一貫性が低い |
| 地域/時間帯単位 | クラスター単位 | サンプルサイズの有効数が減少 |

**バランス確認 (SRM: Sample Ratio Mismatch)**:
- 割り当て比率と実際の比率がカイ二乗検定で有意にずれていないか確認
- SRM検出 → 実装バグの兆候。結果を信頼してはいけない

### 3.4 多重比較補正

**問題**: K個の仮説を同時検定 → 全体の偽陽性率が α×K に膨張

| 補正法 | 式 | 保守性 | 適用場面 |
|---|---|---|---|
| Bonferroni | α' = α/K | 強（保守的） | 少数の独立検定 |
| Holm-Bonferroni | ステップダウン方式 | 中 | Bonferroniよりやや緩い |
| Benjamini-Hochberg (FDR) | False Discovery Rate制御 | 弱 | 多数の検定（探索的） |
| Tukey HSD | 全ペア比較専用 | 中 | ANOVA後の多重比較 |
| Dunnett | 対照群 vs 複数処理群 | 中 | 1つの対照群がある場合 |

### 3.5 ベイジアンA/Bテスト

**頻度主義 vs ベイズ**:

| 観点 | 頻度主義 | ベイズ |
|---|---|---|
| 確率の解釈 | 長期的頻度 | 信念の度合い |
| 結果の表現 | p値、信頼区間 | 事後分布、損失関数 |
| サンプルサイズ | 事前に固定 | いつでものぞき見可能 |
| 早期停止 | 偽陽性率膨張リスク | 自然に対応 |
| 事前知識 | 使わない | 事前分布として組み込む |

**ベイジアン手法の概要**:
- 事前分布 × 尤度 → 事後分布
- 二項データの場合: Beta事前分布 × 二項尤度 → Beta事後分布
- 意思決定: P(A > B) > 閾値（例: 95%）なら A を採用
- 期待損失: 各選択の期待損失を比較

### 3.6 実務活用: A/Bテスト運用

**テスト設計チェックリスト**:
1. 仮説と指標を事前登録（p-hackingは確証バイアスの温床）
2. サンプルサイズ計算 → テスト期間を算出（最低1週間の倍数）
3. AA テスト（同一バージョン）で計測基盤の健全性を確認
4. ランプアップ: 1%→10%→50%と段階的に拡大
5. SRMチェック → 異常があれば原因究明
6. 結果分析: 主要指標の統計検定 + 副次指標の確認
7. セグメント分析: 特定ユーザー群での効果差異
8. 意思決定: 統計的有意 AND 実務的有意（効果量が十分か）

## 4. 時系列分析（実務編）

### 4.1 時系列の基本概念

**時系列の構成要素**:
- **トレンド (T)**: 長期的な上昇/下降傾向
- **季節性 (S)**: 固定周期の繰り返しパターン（日/週/月/年）
- **循環変動 (C)**: 不定期な長期的変動（景気循環）
- **不規則変動 (I)**: ランダムなノイズ

**分解モデル**:
- 加法モデル: Y = T + S + I（変動幅が一定）
- 乗法モデル: Y = T × S × I（変動幅がトレンドに比例）

**定常性 (Stationarity)**:
- 弱定常: 平均と自己共分散が時間に依存しない
- ADF検定 (Augmented Dickey-Fuller): 単位根の有無を検定
- KPSS検定: 定常性の帰無仮説（ADF検定と補完的に使用）
- 非定常 → 差分（1階差分 or 季節差分）で定常化

### 4.2 トレンド・季節性の分解

**STL分解 (Seasonal and Trend decomposition using Loess)**:
- Loess（局所回帰）でトレンドと季節性を柔軟に推定
- ロバスト推定: 外れ値の影響を軽減
- パラメータ: seasonal（季節窓幅）、trend（トレンド窓幅）

```python
from statsmodels.tsa.seasonal import STL
stl = STL(series, period=7, robust=True)
result = stl.fit()
# result.trend, result.seasonal, result.resid
```

### 4.3 予測モデルの使い分け

| モデル | 特徴 | 適用場面 | 限界 |
|---|---|---|---|
| 移動平均 | シンプル、解釈容易 | 短期平滑化、ベースライン | トレンド・季節性を捉えない |
| 指数平滑法 | 直近の観測に重み | 短期予測、Holt-Winters | 外部変数を扱えない |
| ARIMA | Box-Jenkins法 | 単変量、定常過程 | 非線形パターンに弱い |
| SARIMA | ARIMA + 季節項 | 季節性あり | パラメータ同定が難しい |
| Prophet | Facebookが開発 | 休日・イベント対応、欠損に強い | 複雑な非線形に弱い |
| VAR | 多変量AR | 変数間の相互影響 | 次元の呪い |
| 状態空間モデル | カルマンフィルタ | 構造変化、欠損に強い | 計算コスト |
| LightGBM/XGBoost | 機械学習 | 多数の外部変数 | 外挿に弱い、解釈困難 |
| N-BEATS/TFT | 深層学習 | 大規模時系列 | データ量が必要 |

**モデル選択の判断フロー**:
1. 外部変数が重要か → Yes: Prophet / ML / VAR
2. 季節性があるか → Yes: SARIMA / Holt-Winters / Prophet
3. 非線形パターンか → Yes: ML / 深層学習
4. 少数系列か多数系列か → 多数: Global Model（LightGBM / N-BEATS）
5. 解釈性が重要か → Yes: ARIMA / 指数平滑法

### 4.4 異常検知

| 手法 | 原理 | 適用場面 |
|---|---|---|
| 統計的管理図 | μ±3σ（シューハート管理図） | プロセス管理、品質管理 |
| CUSUM | 累積和が閾値を超過 | 平均の微小シフト検出 |
| Isolation Forest | ランダム分割の孤立速度 | 多変量、非正規分布 |
| STL + 残差監視 | 残差の異常スコア | 季節性を除外した異常検出 |
| Prophet異常検知 | 予測区間からの逸脱 | 休日・イベント考慮 |
| Spectral Residual | スペクトル分析の残差 | 周期的データの異常 |

### 4.5 実務活用: 需要予測とKPI監視

**需要予測パイプライン**:
1. EDA: トレンド・季節性・外れ値の確認
2. 特徴量: カレンダー（祝日/曜日）、プロモーション、天気、価格
3. 複数モデル構築: SARIMA + Prophet + LightGBM
4. クロスバリデーション: 時系列用CV（Expanding Window or Sliding Window）
5. アンサンブル: 加重平均 or スタッキング
6. 評価: MAPE, WAPE, RMSSE（M5 Competition標準）
7. 監視: 予測誤差のドリフト監視

**KPI監視**:
- ベースラインモデル（Prophet等）で「あるべき値」を予測
- 実績が予測区間外 → アラート
- 曜日・祝日効果を考慮しないと偽陽性が多発

## 5. テキスト分析

### 5.1 テキスト前処理

**処理ステップ**:
1. **トークン化**: 文→単語/サブワードに分割
2. **正規化**: 小文字化、Unicode正規化、表記ゆれ統一
3. **ストップワード除去**: 機能語（の、は、が等）の除外
4. **ステミング/レンマタイゼーション**: 語幹への変換
5. **形態素解析**（日本語）: MeCab, Janome, SudachiでN-gram/形態素分割

### 5.2 テキスト特徴量

**TF-IDF (Term Frequency - Inverse Document Frequency)**:
```
TF(t,d) = (文書d中の単語tの出現回数) / (文書dの全単語数)
IDF(t) = log(全文書数 / 単語tを含む文書数)
TF-IDF(t,d) = TF(t,d) × IDF(t)
```
- 高TF-IDF: その文書に特徴的な重要語
- 低TF-IDF: ストップワード（多くの文書に出現）

**Bag of Words (BoW)**: 単語の出現回数ベクトル。語順情報は失われる
**N-gram**: 連続するN個の単語/文字。バイグラム（N=2）で共起を捉える

### 5.3 トピックモデル

**LDA (Latent Dirichlet Allocation)**:
- 文書はトピックの混合、トピックは単語の混合
- 生成モデル: 各文書のトピック分布 → 各トピックの単語分布からサンプリング
- ハイパーパラメータ: α（文書のトピック集中度）、β（トピックの単語集中度）
- トピック数の決定: Coherenceスコア（C_v）の最大化 or Perplexityの肘

**NMF (Non-negative Matrix Factorization)**:
- TF-IDF行列を非負制約付きで分解 → トピック×単語行列
- LDAより高速、解釈性が高い場合がある

### 5.4 分散表現・深層学習

**Word2Vec**:
- CBOW: 周辺語から中心語を予測
- Skip-gram: 中心語から周辺語を予測
- 「王 - 男 + 女 ≈ 女王」のような意味的演算が可能
- 固定ベクトル（同音異義語を区別しない）

**BERT (Bidirectional Encoder Representations from Transformers)**:
- 双方向コンテキスト: 前後の文脈から単語表現を生成
- 事前学習（MLM + NSP）→ ファインチューニング
- 文脈依存ベクトル: 同じ単語でも文脈で異なる表現
- 日本語: cl-tohoku/bert-base-japanese, nlp-waseda/roberta 等

**Sentence Transformers**:
- 文レベルの埋め込みベクトル生成
- 文類似度計算、意味検索、クラスタリングに利用

### 5.5 感情分析

| 手法 | 説明 | 精度 |
|---|---|---|
| 辞書ベース | 極性辞書（ポジ/ネガ/ニュートラル）で単語スコア集計 | 低〜中 |
| 機械学習 | TF-IDF + SVM/LogisticRegression | 中 |
| 深層学習 | BERT + 分類ヘッド | 高 |
| LLM | GPT/Claude でゼロショット/フューショット | 高（コスト大） |

**アスペクトベース感情分析**: 「料理は美味しいがサービスが悪い」→ 料理:positive, サービス:negative

### 5.6 実務活用: VOC分析・文書分類

**VOC (Voice of Customer) 分析パイプライン**:
1. データ収集: レビュー、問い合わせ、SNS
2. 前処理: クレンジング、形態素解析
3. トピック抽出: LDA / BERTopic でカテゴリ自動分類
4. 感情分析: 各トピックのポジ/ネガ傾向
5. トレンド分析: トピック×感情の時系列推移
6. アクション: 頻出ネガティブトピック → 優先改善項目

**文書分類**:
- **ゼロショット分類**: LLMに分類カテゴリを与えて推論（ラベルデータ不要）
- **フューショット分類**: 少数の例を与えてLLMに推論させる
- **ファインチューニング**: BERTを分類タスクで追加学習（高精度、データ必要）
