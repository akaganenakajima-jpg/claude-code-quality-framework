## 1. ニューラルネットワークの数理

### 1.1 パーセプトロン

単層パーセプトロンは線形分離可能な問題のみ解ける。

```
y = step(w^T x + b)
```

- **限界**: XOR問題は単層では解けない（Minsky & Papert, 1969）
- **解決**: 多層パーセプトロン（MLP）で非線形分離が可能に

### 1.2 多層パーセプトロン (MLP)

```
h₁ = f₁(W₁x + b₁)
h₂ = f₂(W₂h₁ + b₂)
y  = f₃(W₃h₂ + b₃)
```

- 各層の出力 = 活性化関数(線形変換)
- 層数が深いほど表現力は上がるが、学習が困難になる（勾配消失）

### 1.3 活性化関数

#### Sigmoid
```
σ(x) = 1 / (1 + exp(-x))
σ'(x) = σ(x)(1 - σ(x))
```
- 出力範囲: (0, 1)
- **問題**: 飽和領域で勾配≈0（勾配消失）、出力が非ゼロ中心
- **用途**: 出力層（二値分類の確率出力）、ゲート機構（LSTM）

#### tanh
```
tanh(x) = (exp(x) - exp(-x)) / (exp(x) + exp(-x))
tanh'(x) = 1 - tanh²(x)
```
- 出力範囲: (-1, 1) — ゼロ中心
- **問題**: 飽和領域で勾配消失（sigmoidと同様）
- **用途**: RNNの隠れ状態、sigmoidよりは一般的に良い

#### ReLU (Rectified Linear Unit)
```
ReLU(x) = max(0, x)
ReLU'(x) = 1 (x > 0), 0 (x ≤ 0)
```
- **利点**: 計算が高速、勾配消失しにくい、スパース活性化
- **問題**: Dying ReLU（負の入力で勾配が永久に0）
- **用途**: 中間層のデフォルト選択

#### Leaky ReLU
```
LeakyReLU(x) = x (x > 0), αx (x ≤ 0)   [α = 0.01 が典型]
LeakyReLU'(x) = 1 (x > 0), α (x ≤ 0)
```
- Dying ReLU問題を緩和（負領域にも小さな勾配）
- **PReLU**: αを学習パラメータにした変種

#### GELU (Gaussian Error Linear Unit)
```
GELU(x) = x · Φ(x)   [Φ: 標準正規分布のCDF]
≈ 0.5x(1 + tanh(√(2/π)(x + 0.044715x³)))
```
- 確率的にニューロンを活性化（ドロップアウトに似た効果）
- **用途**: Transformer系モデル（BERT, GPT）の標準

#### Swish (SiLU)
```
Swish(x) = x · σ(βx)   [β=1 が一般的]
```
- 滑らかなReLU（微分が連続）
- GELUと性能がほぼ同等
- **用途**: EfficientNet、モバイルモデル

#### Softmax
```
Softmax(xᵢ) = exp(xᵢ) / Σⱼ exp(xⱼ)
```
- 出力の総和 = 1（確率分布として解釈可能）
- **数値安定性**: `exp(xᵢ - max(x))` で計算（オーバーフロー防止）
- **温度パラメータ**: `Softmax(xᵢ/T)` — T大→均一、T小→尖鋭
- **用途**: 多クラス分類の出力層

#### 活性化関数の選択指針

| 層 | 推奨 | 理由 |
|----|------|------|
| 中間層（CNN/MLP） | ReLU / Leaky ReLU | 計算効率、勾配消失防止 |
| 中間層（Transformer） | GELU / Swish | 滑らか、経験的に高性能 |
| 出力層（二値分類） | Sigmoid | (0,1)の確率出力 |
| 出力層（多クラス） | Softmax | 確率分布 |
| 出力層（回帰） | なし（恒等関数） | 任意の実数値 |
| ゲート機構 | Sigmoid | (0,1)のゲート値 |

### 1.4 損失関数

#### 平均二乗誤差 (MSE)
```
L = (1/n) Σ (yᵢ - ŷᵢ)²
∂L/∂ŷᵢ = (2/n)(ŷᵢ - yᵢ)
```
- 外れ値に敏感（二乗で増幅）
- **用途**: 回帰問題

#### 平均絶対誤差 (MAE)
```
L = (1/n) Σ |yᵢ - ŷᵢ|
∂L/∂ŷᵢ = (1/n) sign(ŷᵢ - yᵢ)
```
- 外れ値にロバスト（MSEより）
- **問題**: 0で微分不可能
- **用途**: 外れ値が多い回帰問題

#### Huber損失（Smooth L1）
```
L = 0.5(y-ŷ)²          (|y-ŷ| ≤ δ)
    δ|y-ŷ| - 0.5δ²     (|y-ŷ| > δ)
```
- MSEとMAEのハイブリッド（小さい誤差はMSE、大きい誤差はMAE）
- **用途**: 物体検出のバウンディングボックス回帰

#### 二値交差エントロピー (Binary Cross-Entropy)
```
L = -(1/n) Σ [yᵢ log(ŷᵢ) + (1-yᵢ) log(1-ŷᵢ)]
∂L/∂ŷᵢ = -(1/n)(yᵢ/ŷᵢ - (1-yᵢ)/(1-ŷᵢ))
```
- **用途**: 二値分類

#### カテゴリカル交差エントロピー
```
L = -(1/n) Σᵢ Σⱼ yᵢⱼ log(ŷᵢⱼ)
```
- Softmax + Cross-Entropy の組合せが数値的に安定
  - `log_softmax` → `nll_loss` のパイプライン
- **用途**: 多クラス分類

#### Focal Loss
```
FL = -αₜ(1 - pₜ)^γ log(pₜ)
```
- γ=0 なら通常のCross-Entropy
- γ>0 で簡単な例のlossを下げ、難しい例に集中
- **用途**: クラス不均衡問題（物体検出のRetinaNet）

#### Hinge Loss
```
L = max(0, 1 - y·ŷ)   [y ∈ {-1, +1}]
```
- マージン最大化（SVMの損失関数）
- **用途**: SVM、分類問題の正則化項

### 1.5 普遍近似定理 (Universal Approximation Theorem)

> 十分な数の隠れユニットを持つ1隠れ層のNNは、任意の連続関数を任意の精度で近似できる（Cybenko, 1989; Hornik, 1991）

- **注意**: 「近似できる」は「学習できる」とは異なる
  - 必要なユニット数が指数的に大きくなる可能性がある
  - 深いネットワークの方が効率的に表現できる（深さの利点）
- **深さの定理**: 深さkの回路で表現可能な関数を、深さk-1では指数的に多いユニットが必要（Eldan & Shamir, 2016）

### 1.6 実務活用: アーキテクチャ選択の根拠

- 分類 → Softmax + Cross-Entropy（ほぼ例外なし）
- 回帰 → 恒等出力 + MSE（外れ値多ければHuber）
- 不均衡分類 → Focal Loss + クラス重み
- 中間層 → ReLU（まず試す）→ GELU（Transformer）
- 深いネットワーク → He初期化 + 残差結合 + バッチ正規化

## 2. 誤差逆伝播法

### 2.1 計算グラフ

ニューラルネットワークの演算を有向非巡回グラフ（DAG）で表現する。

```
[前向き計算 (Forward Pass)]
x → [Linear] → z → [ReLU] → a → [Linear] → ŷ → [Loss] → L

[後ろ向き計算 (Backward Pass)]
∂L/∂L=1 → ∂L/∂ŷ → ∂L/∂a → ∂L/∂z → ∂L/∂W, ∂L/∂b
```

- 前向き: 入力→出力の計算、中間値を保存
- 後ろ向き: 出力→入力の勾配計算（逆順）

### 2.2 連鎖律 (Chain Rule) の適用

```
∂L/∂W₁ = ∂L/∂ŷ · ∂ŷ/∂h₂ · ∂h₂/∂h₁ · ∂h₁/∂W₁
```

具体例（2層MLP、Sigmoid活性化、MSE損失）:
```
前向き:
  z₁ = W₁x + b₁
  h₁ = σ(z₁)
  z₂ = W₂h₁ + b₂
  ŷ  = σ(z₂)
  L  = (1/2)(y - ŷ)²

後ろ向き:
  δ₂ = ∂L/∂z₂ = (ŷ - y) · σ'(z₂)
  ∂L/∂W₂ = δ₂ · h₁ᵀ
  ∂L/∂b₂ = δ₂
  δ₁ = ∂L/∂z₁ = (W₂ᵀ δ₂) · σ'(z₁)
  ∂L/∂W₁ = δ₁ · xᵀ
  ∂L/∂b₁ = δ₁
```

### 2.3 自動微分 (Automatic Differentiation)

手動微分や数値微分（有限差分）ではなく、計算グラフを利用。

| 方式 | 計算方向 | 計算量 | 用途 |
|------|---------|--------|------|
| Forward Mode | 入力→出力 | O(入力次元 × 計算量) | 入力少・出力多 |
| Reverse Mode | 出力→入力 | O(出力次元 × 計算量) | 入力多・出力少（DL標準） |

- **PyTorch**: Reverse Mode（`loss.backward()` で全パラメータの勾配を一度に計算）
- **JAX**: Forward/Reverse両方対応（`jax.grad`, `jax.jacfwd`）

### 2.4 勾配消失・勾配爆発問題

**勾配消失 (Vanishing Gradient)**:
- 層が深いほど、初期層の勾配が指数的に小さくなる
- 原因: sigmoid/tanhの飽和領域（|σ'| < 0.25）
- 各層で σ'(z) を掛けるため: `|∂L/∂W₁| ∝ Π σ'(zₖ)` → 0 に収束

**勾配爆発 (Exploding Gradient)**:
- 勾配が指数的に大きくなる（特にRNN）
- 原因: 重み行列の最大特異値 > 1 の繰り返し乗算

**解決策**:

| 問題 | 解決策 | 詳細 |
|------|--------|------|
| 勾配消失 | ReLU活性化 | 正領域で勾配=1（消失しない） |
| 勾配消失 | 残差結合 (Skip Connection) | `h = F(x) + x` で勾配のショートカット |
| 勾配消失 | バッチ正規化 | 中間出力を正規化し活性化の飽和を防ぐ |
| 勾配消失 | LSTM/GRUのゲート | セル状態が勾配を直接伝播 |
| 勾配爆発 | 勾配クリッピング | `‖g‖ > θ → g ← θg/‖g‖` |
| 勾配爆発 | 適切な初期化 | Xavier/He初期化で分散を制御 |
| 両方 | 学習率の適切な設定 | 大きすぎ→発散、小さすぎ→収束遅い |

### 2.5 実務活用: PyTorchの autograd 理解

```python
# 計算グラフの構築と勾配計算
x = torch.tensor([1.0], requires_grad=True)
y = x ** 2 + 2 * x
y.backward()        # dy/dx = 2x + 2 = 4
print(x.grad)       # tensor([4.])

# 勾配の蓄積に注意（明示的にゼロ化が必要）
optimizer.zero_grad()
loss.backward()
optimizer.step()

# detach() で計算グラフから切り離し（メモリ節約）
with torch.no_grad():
    val_loss = model(val_data)
```

## 3. 重みの初期化

### 3.1 初期化の重要性

- 全て0で初期化 → 全ニューロンが同一出力 → 対称性が破れない
- ランダムでも分散が不適切 → 勾配消失/爆発
- 目標: 各層の出力分散を一定に保つ（分散の伝播）

### 3.2 Xavier/Glorot 初期化

sigmoid/tanh用（線形領域で動作する前提）:
```
W ~ U(-√(6/(nᵢₙ+nₒᵤₜ)), √(6/(nᵢₙ+nₒᵤₜ)))   [一様分布]
W ~ N(0, 2/(nᵢₙ+nₒᵤₜ))                           [正規分布]
```

- 導出: `Var(output) = Var(input)` を満たす条件
  - 前向き: `Var(y) = nᵢₙ · Var(W) · Var(x)` → `Var(W) = 1/nᵢₙ`
  - 後ろ向き: `Var(∂L/∂x) = nₒᵤₜ · Var(W) · Var(∂L/∂y)` → `Var(W) = 1/nₒᵤₜ`
  - 妥協: `Var(W) = 2/(nᵢₙ + nₒᵤₜ)`

### 3.3 He 初期化

ReLU用（ReLUは半分の出力を0にするため分散が半減）:
```
W ~ N(0, 2/nᵢₙ)
```

- ReLU: `E[ReLU(x)²] = Var(x)/2` → 分散が半分になるので係数2で補正
- Leaky ReLU: `Var(W) = 2/((1+α²)nᵢₙ)`

### 3.4 直交初期化 (Orthogonal Initialization)

```
W = QR分解でQを取得し、適切にスケーリング
```

- 特異値がすべて1 → 勾配の大きさが層を通して不変
- **用途**: RNNの再帰重み（勾配消失/爆発を緩和）

### 3.5 その他の初期化

- **LeCun初期化**: `Var(W) = 1/nᵢₙ`（SELUと組合せ）
- **LSUV (Layer-sequential Unit-Variance)**: 各層の出力分散を1に調整
- **Transfer Learning**: 事前学習済み重みで初期化（最も実用的）

### 3.6 初期化の選択指針

| 活性化関数 | 推奨初期化 | PyTorch |
|-----------|----------|---------|
| Sigmoid/tanh | Xavier | `nn.init.xavier_uniform_` |
| ReLU/Leaky ReLU | He (Kaiming) | `nn.init.kaiming_normal_` |
| SELU | LeCun | `nn.init.lecun_normal_` (手動) |
| RNNの再帰重み | 直交 | `nn.init.orthogonal_` |
| Transformer | Xavier (慣例) | ライブラリ内蔵 |

### 3.7 実務活用: 深いネットワークの安定学習

- PyTorchのデフォルト初期化は `kaiming_uniform_`（ReLU想定）
- カスタムモデルで活性化関数を変えた場合、初期化も変更が必要
- 学習が発散する場合、まず初期化と学習率を疑う
- 転移学習ではランダム初期化の層のみ学習率を高く設定（差分学習率）

## 4. 正則化

### 4.1 過学習とバイアス-バリアンストレードオフ

```
総誤差 = バイアス² + バリアンス + ノイズ
```

- **バイアス**: モデルの表現力不足による誤差（過小適合）
- **バリアンス**: データに敏感すぎる誤差（過学習）
- 正則化はバリアンスを減らし、汎化性能を上げる

### 4.2 L1/L2 正則化

#### L2正則化 (Ridge / Weight Decay)
```
L_total = L_data + (λ/2) Σ wᵢ²
∂L_total/∂w = ∂L_data/∂w + λw
```
- 重みを0に近づける（完全に0にはならない）
- 勾配更新: `w ← w(1 - ηλ) - η·∂L_data/∂w`

#### L1正則化 (Lasso)
```
L_total = L_data + λ Σ |wᵢ|
```
- スパースな重み（多くの重みが完全に0になる）
- **用途**: 特徴選択（不要な入力の重みを0に）

#### Elastic Net
```
L_total = L_data + λ₁Σ|wᵢ| + (λ₂/2)Σwᵢ²
```
- L1とL2の組合せ

#### Weight Decay vs L2正則化
- SGD: 同等
- Adam: **異なる** — AdamW は Weight Decay を勾配計算と分離（Loshchilov & Hutter, 2019）
  - Adam + L2: 適応的学習率が正則化効果を打ち消す
  - AdamW: `w ← w(1 - ηλ) - η·m̂/(√v̂+ε)` — 正しい実装

### 4.3 ドロップアウト (Dropout)

```
学習時: hᵢ = hᵢ · mᵢ / (1-p)   [mᵢ ~ Bernoulli(1-p)]
推論時: hᵢ = hᵢ               [スケーリング不要（Inverted Dropout）]
```

- 確率pでニューロンを無効化（p=0.5が典型、最終層近くは0.2-0.3）
- **解釈1**: アンサンブル学習の近似（2^n個のサブネットの平均）
- **解釈2**: 共適応（co-adaptation）の防止
- **注意**: バッチ正規化との併用は効果が相殺されることがある

#### DropConnect
- 重みレベルでドロップ（ニューロンではなく接続を切る）
- ドロップアウトの一般化

#### Spatial Dropout
- CNNでチャネル単位でドロップ（空間的相関を考慮）

### 4.4 バッチ正規化 (Batch Normalization)

```
学習時:
  μ_B = (1/m) Σ xᵢ
  σ²_B = (1/m) Σ (xᵢ - μ_B)²
  x̂ᵢ = (xᵢ - μ_B) / √(σ²_B + ε)
  yᵢ = γx̂ᵢ + β   [γ, β は学習パラメータ]

推論時:
  移動平均の μ, σ² を使用（バッチ統計量ではない）
```

- **内部共変量シフト**: 各層の入力分布が学習中に変化する問題（元論文の主張）
  - 後の研究: 実際には損失ランドスケープの平滑化が主効果（Santurkar et al., 2018）
- **効果**: 学習の高速化、高い学習率の許容、正則化効果
- **配置**: Conv/Linear の**後**、活性化関数の**前**が一般的
  - `Conv → BN → ReLU`（最も一般的）
- **バッチサイズ依存**: 小バッチ（<16）では統計量が不安定

### 4.5 他の正規化手法

| 手法 | 正規化の軸 | バッチサイズ依存 | 主な用途 |
|------|-----------|----------------|---------|
| Batch Norm | バッチ方向 | あり | CNN |
| Layer Norm | 特徴量方向 | なし | Transformer, RNN |
| Instance Norm | チャネル内空間方向 | なし | スタイル変換 |
| Group Norm | チャネルをグループ分割 | なし | 小バッチCNN |

```
入力テンソル: (N, C, H, W)  [バッチ, チャネル, 高さ, 幅]

Batch Norm:    (N, -, H, W) で正規化 → C個の統計量
Layer Norm:    (-, C, H, W) で正規化 → N個の統計量
Instance Norm: (-, -, H, W) で正規化 → N×C個の統計量
Group Norm:    (-, G, C/G, H, W) で正規化 → N×G個の統計量
```

- **Layer Norm**: Transformerの標準（Attention/FFNの前後に配置）
  - **Pre-LN**: LN→Attention/FFN→残差結合（学習が安定、GPT-2以降の主流）
  - **Post-LN**: Attention/FFN→残差結合→LN（元のTransformer、学習不安定）
- **RMSNorm**: Layer Normから平均の引き算を省略（LLaMAで採用、計算効率）

### 4.6 データ拡張 (Data Augmentation)

画像:
| 手法 | 説明 | 効果 |
|------|------|------|
| 水平反転 | 左右反転 | 位置不変性 |
| 回転 | ±10-30度 | 角度不変性 |
| 色変換 | 明度/彩度/色相の変化 | 照明条件への頑健性 |
| ランダムクロップ | 画像の一部を切り出し | 位置不変性 |
| Cutout | 画像の矩形領域をマスク | オクルージョン耐性 |
| MixUp | 2画像の線形結合 `x̃=λx₁+(1-λ)x₂` | 決定境界の平滑化 |
| CutMix | 一方の矩形領域を他方に貼付 | MixUp+Cutoutの効果 |
| RandAugment | N個の変換をランダムに適用 | 自動化（探索不要） |
| AutoAugment | 強化学習で最適な拡張方策を探索 | データセット特化 |

テキスト:
- 同義語置換、ランダム挿入/削除/入替
- Back Translation（英→日→英）
- EDA (Easy Data Augmentation)

### 4.7 早期停止 (Early Stopping)

```
if val_loss が patience エポック改善しない:
    学習を停止し、最良のモデルを復元
```

- 暗黙の正則化（学習時間≈モデルの複雑さを制限）
- `patience` と `min_delta` の設定が重要

### 4.8 実務活用: 過学習防止の戦略選択

```
過学習の兆候: train_loss↓ val_loss↑ → 以下の順で対策

1. データ拡張（最もコスト効果が高い）
2. Weight Decay（ほぼ常に有効）
3. ドロップアウト（FC層に効果大）
4. 早期停止（簡単で確実）
5. モデルの縮小（最終手段）

過小適合の兆候: train_loss↓しない → モデルの拡大/学習率調整
```

## 5. 評価指標

### 5.1 分類指標

#### 混同行列 (Confusion Matrix)

```
              予測
              P    N
実際  P  [  TP | FN  ]
      N  [  FP | TN  ]
```

#### 基本指標

```
Accuracy  = (TP + TN) / (TP + TN + FP + FN)
Precision = TP / (TP + FP)   — 予測Pのうち実際にP（適合率）
Recall    = TP / (TP + FN)   — 実際Pのうち予測でP（再現率）
F1        = 2 · Precision · Recall / (Precision + Recall)  — 調和平均
Fβ        = (1+β²) · P·R / (β²·P + R)  — β>1でRecall重視、β<1でPrecision重視
```

- **Accuracy**: クラス不均衡時に無意味（99%が負例なら「全て負」で99%）
- **Precision重視**: スパム判定（誤ってスパム判定=コスト高）
- **Recall重視**: がん検出（見逃し=コスト高）

#### マルチクラスの平均化

| 方式 | 計算 | 用途 |
|------|------|------|
| Macro | クラス毎に計算→平均 | 全クラス等重要 |
| Micro | 全体のTP/FP/FNで計算 | サンプル数重視 |
| Weighted | クラス毎に計算→サンプル数で加重平均 | 不均衡データ |

#### AUC-ROC

```
ROC曲線: x軸=FPR, y軸=TPR（閾値を変化させてプロット）
FPR = FP / (FP + TN)
TPR = Recall = TP / (TP + FN)
AUC = ROC曲線下面積（0.5=ランダム、1.0=完璧）
```

- 閾値に依存しない総合評価
- クラス不均衡時は PR-AUC の方が適切

#### PR-AUC (Precision-Recall AUC)

```
PR曲線: x軸=Recall, y軸=Precision
```

- 正例が少ない（不均衡）データに適切
- ROC-AUCでは負例が大量→FPRが低くなり高AUCになりやすい

#### Log Loss (Cross-Entropy Loss)

```
LogLoss = -(1/n) Σ [yᵢ log(pᵢ) + (1-yᵢ) log(1-pᵢ)]
```

- 確率予測の品質を評価（確信度も評価される）
- Calibration（確率値の信頼性）を反映

### 5.2 回帰指標

```
MSE  = (1/n) Σ (yᵢ - ŷᵢ)²          — 外れ値に敏感
RMSE = √MSE                          — 元のスケール
MAE  = (1/n) Σ |yᵢ - ŷᵢ|            — ロバスト
MAPE = (100/n) Σ |yᵢ - ŷᵢ|/|yᵢ|     — 相対誤差（yᵢ=0で不可）
R²   = 1 - Σ(yᵢ-ŷᵢ)² / Σ(yᵢ-ȳ)²   — 決定係数（1が最良、負もあり得る）
```

### 5.3 生成モデル指標

| 指標 | 計算 | 特徴 |
|------|------|------|
| FID (Frechet Inception Distance) | Inception-v3の特徴量空間でのフレシェ距離 | 低いほど良い、品質+多様性 |
| IS (Inception Score) | `exp(E[KL(p(y|x) ‖ p(y))])` | 高いほど良い、品質のみ |
| LPIPS | 学習済みCNNの中間特徴量の距離 | 人間の知覚に近い |
| SSIM | 構造的類似性 | 画像の歪み評価 |

### 5.4 NLP指標

| 指標 | 計算 | 用途 |
|------|------|------|
| BLEU | n-gram精度の幾何平均 + Brevity Penalty | 機械翻訳 |
| ROUGE-N | n-gramのRecall | 文書要約 |
| ROUGE-L | 最長共通部分列 (LCS) | 文書要約 |
| BERTScore | BERTエンベディングのコサイン類似度 | 意味的類似性 |
| Perplexity | `exp(-(1/N)Σ log p(wᵢ))` | 言語モデル（低いほど良い） |

### 5.5 実務活用: タスクに応じた指標選択

| タスク | 主要指標 | 補助指標 |
|--------|---------|---------|
| 二値分類（均衡） | Accuracy, F1 | AUC-ROC |
| 二値分類（不均衡） | PR-AUC, F1 | Recall@k |
| 多クラス分類 | Macro F1 | Confusion Matrix |
| 回帰 | RMSE, MAE | R², MAPE |
| 物体検出 | mAP@IoU=0.5 | mAP@[0.5:0.95] |
| セグメンテーション | mIoU | Dice |
| 画像生成 | FID | IS, LPIPS |
| 機械翻訳 | BLEU | BERTScore |
| 文書要約 | ROUGE-L | BERTScore |
| 言語モデル | Perplexity | BLEU (下流タスク) |

## 付録: 試験頻出の計算パターン

### A. Softmax + Cross-Entropy の勾配（超頻出）

```
L = -Σ yⱼ log(softmax(zⱼ))

∂L/∂zᵢ = softmax(zᵢ) - yᵢ = pᵢ - yᵢ
```
- 驚くほどシンプル（予測確率 - 正解ラベル）

### B. Sigmoid + Binary CE の勾配

```
L = -[y log σ(z) + (1-y) log(1-σ(z))]

∂L/∂z = σ(z) - y
```
- Softmax + CEの二値版と同じ形

### C. バッチ正規化の逆伝播

```
∂L/∂xᵢ = (γ/√(σ²+ε)) · [∂L/∂x̂ᵢ - (1/m)Σ∂L/∂x̂ⱼ - x̂ᵢ·(1/m)Σ∂L/∂x̂ⱼ·x̂ⱼ]
```
- 試験ではステップごとの導出が問われる
