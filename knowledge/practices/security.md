# セキュリティプラクティス

> **プロジェクト CLAUDE.md の指定が優先**。ここは汎用デフォルト。
> IPA知識ベース参照: `ipa/ap.md` §ゼロトラスト, `ipa/au.md` §監査証跡

## 1. 入力検証（ゼロトラスト原則）

- ユーザー/API/外部サービスからのデータは**全て信頼しない**
- 検証項目: 型・範囲・形式・長さ・文字種
- バリデーションはできるだけ入口（APIハンドラ）で実施
- ホワイトリスト方式を優先（許可するものを定義）

## 2. 認証・認可

- 管理系エンドポイントには必ず認証を設ける
- トークン/APIキーはリクエストヘッダーで受け渡し（URLパラメータ禁止）
- 最小権限の原則: 必要最低限の権限のみ付与
- セッション管理: 有効期限・無効化・再認証の仕組みを設ける

## 3. 秘密情報管理

- APIキー・パスワード・トークンをコードに直書き**禁止**
- 環境変数 or 秘密管理ツール（Wrangler Secret, Vault等）を使う
- `.env` ファイルは `.gitignore` に必ず含める
- ログ・エラーメッセージに秘密情報を出力しない

## 4. エラー情報の保護

- 本番環境でスタックトレースを外部に返さない
- 内部パス・DB構造・処理詳細をエラーレスポンスに含めない
- エラーコード + ユーザー向けメッセージのみ返す
- 詳細は内部ログにのみ記録

## 5. 依存パッケージの安全性

- 定期的に `npm audit` / `pip audit` で脆弱性チェック
- 高リスク脆弱性（Critical/High）は即対応
- 依存パッケージのバージョンを固定（lockfile管理）
- 不要な依存は削除する

## 6. ログの安全性（`ipa/au.md` §監査証跡）

- 個人情報（氏名・住所・電話番号等）をログに出力しない
- パスワード・トークン・APIキーをログに出力しない
- ログには操作者・日時・操作内容・結果を記録（追跡可能性確保）
- ログの改ざん防止策を講じる（append-only, 外部保管等）

## 7. インシデント対応

- 発生時: 検知 → 影響範囲特定 → 封じ込め → 復旧 → 原因分析
- 復旧後: 原因・対応内容を記録してから監視再開
- 再発防止: 根本原因に対する是正処置 + 水平展開
