# 選択設計 — UX心理学リファレンス

> ユーザーの意思決定を支援し、望ましい選択へ自然に導くための心理学法則。
> 各セクション末尾に「実務活用」を収録。

---

## §1. アンカー効果（Anchoring Effect）

> 最初に提示された数値や情報が「錨」となり、後続の判断を無意識に引き寄せる。

### 1.1 心理学的メカニズム

Tversky & Kahneman（1974）の研究により実証された認知バイアス。人間は不確実な状況で数値を推定する際、最初に提示された数値（アンカー）に引きずられる。この効果は、提示されたアンカーが明らかに無関係な場合でも発生する。

メカニズムは「不十分な調整（insufficient adjustment）」で説明される。人はアンカーを起点として調整を行うが、その調整は常に不十分であり、最終的な判断はアンカーに近い値に偏る。アンカー効果は専門家であっても回避が難しく、意識していても影響を受ける点が特徴的である。

### 1.2 UIへの適用

| パターン | 実装例 | 効果 |
|---|---|---|
| 価格アンカー | 値引き前価格（¥12,000）を取り消し線で表示し、現在価格（¥7,800）を強調 | 割引感を増幅し、現在価格を「お得」と認知させる |
| プラン順序 | 最上位プラン（Enterprise ¥50,000/月）を最初に表示し、中間プラン（Pro ¥9,800/月）を続けて配置 | 中間プランの価格妥当性が向上する |
| デフォルト数量 | ECサイトの数量セレクタをデフォルト「1」ではなく「3」に設定 | 平均購入数量が増加する |
| 寄付金額プリセット | 寄付フォームに「¥1,000 / ¥3,000 / ¥5,000 / ¥10,000」を提示 | 最低額がアンカーとなり平均寄付額が上昇する |
| 比較基準の提示 | 「1日あたり約33円」と月額を日割り表示 | 金額の絶対値を小さく知覚させる |

### 1.3 Apple HIG との接点

- **サブスクリプション表示ガイドライン**: App Store の価格表示では、月額と年額を並列に提示し、年額プランの「月あたり換算額」を表示することで、月額がアンカーとして機能する設計を推奨している
- **In-App Purchase の価格表示**: 価格帯を明示し、ユーザーが自身の支出を適切に判断できる情報提供を求めている

### 1.4 アンチパターン

- **虚偽の元値表示**: 実際に販売実績のない高額な「通常価格」を掲示する行為は景品表示法（有利誤認）に抵触する。二重価格表示には「過去8週間のうち4週間以上の販売実績」が必要
- **非現実的アンカー**: 明らかに不合理な比較対象（例: 「本来なら100万円の価値」）を提示すると、ユーザーの信頼を損なう。アンカーは信憑性のある範囲内に設定すべきである
- **アンカーの乱用**: 画面上に複数のアンカーが競合すると認知負荷が増大し、いずれのアンカーも効果を失う

### 1.5 実務活用

- 価格設計時は「ユーザーが最初に目にする数値」を意識的にコントロールする
- A/Bテストでアンカー値を変動させ、CVR・客単価への影響を計測する
- 法的リスク（景品表示法・特定商取引法）を法務チームと確認してから実装する
- ダッシュボード設計では、KPI の目標値をアンカーとして表示し、現在値との差分を直感的に把握させる

→ 関連: `choice.md` §2（おとり効果）, `impression.md` §2（ハロー効果）

## §2. おとり効果（Decoy Effect）

> 明らかに劣った第三の選択肢を加えることで、特定の選択肢の魅力を相対的に高める。

### 2.1 心理学的メカニズム

Huber, Payne & Puto（1982）が発見した「非対称優越性効果（asymmetric dominance effect）」。選択肢AとBの間で迷っている消費者に、Aには劣るがBとは比較しにくい選択肢A'（おとり）を追加すると、Aの選択率が有意に上昇する。

このメカニズムは「比較の容易性」に基づく。人間は絶対評価よりも相対評価を好み、明確に優劣がつく比較対象があると判断が容易になる。おとりの存在により、ターゲット選択肢が「合理的に優れている」という正当化理由を得られるため、選択に対する自信も高まる。

### 2.2 UIへの適用

| パターン | 実装例 | 効果 |
|---|---|---|
| 3段階プライシング | Basic ¥980, Pro ¥2,980, Enterprise ¥2,780（Proより安いが機能が大幅に少ない）→ Proが最適解に見える | ターゲットプランへの誘導率向上 |
| サイズ比較 | S: 300ml ¥200, M: 500ml ¥350, L: 480ml ¥340 → Mの価値が際立つ | 中間サイズの選択率向上 |
| 機能比較テーブル | 3プランの機能一覧で、おとりプランの欠損を視覚的に明示 | 差異の認知を促進 |
| バンドル設計 | 単品A ¥5,000, 単品B ¥3,000, セットA+B ¥6,000 → 単品Aがおとりとなりセットが選ばれやすい | バンドル選択率の向上 |

### 2.3 Apple HIG との接点

- **比較レイアウト**: HIG はプラン比較において「明確で公正な比較」を推奨しており、各プランの差異が一目で分かるグリッドレイアウトを指針としている
- **選択肢の数**: HIG は選択肢を最小限に抑えることを原則とするが、3つの選択肢による比較は認知的に許容範囲とされている

### 2.4 アンチパターン

- **露骨な操作**: おとりの劣性が明らかすぎると、ユーザーは操作されていると感じ反発する（心理的リアクタンス）
- **選択肢過多**: おとりを含めて4つ以上の選択肢を並べると決断疲れが発生し、逆に選択回避（choice avoidance）を引き起こす
- **おとりの品質不足**: おとりとして配置した選択肢が「存在する意味がない」と認識されると、サービス全体の信頼性に疑問が生じる

### 2.5 実務活用

- プライシングページでは3段階構成を基本とし、中間プランをターゲットに設定する
- おとりの設計は「価格と機能の2軸」で非対称優越性を作ることを意識する
- 定期的にプラン構成のA/Bテストを実施し、おとり効果の強度を検証する
- 法的にはおとり自体は合法だが、選択できない「架空プラン」は不当表示に該当する可能性がある

→ 関連: `choice.md` §1（アンカー効果）, `choice.md` §7（決断疲れ）

## §3. フレーミング効果（Framing Effect）

> 同じ情報でも提示の「枠組み（フレーム）」を変えるだけで、ユーザーの判断が大きく変わる。

### 3.1 心理学的メカニズム

Tversky & Kahneman（1981）による「アジアの疫病問題」で実証された。600人が罹患する疫病への対策として、「200人が助かる」（ポジティブフレーム）と「400人が死亡する」（ネガティブフレーム）では、客観的に同じ結果にもかかわらず選好が逆転する。

ポジティブフレームではリスク回避的選択（確実な利得）が好まれ、ネガティブフレームではリスク追求的選択（ギャンブル）が好まれる。この非対称性はプロスペクト理論の価値関数で説明される。利得領域は凹関数（リスク回避）、損失領域は凸関数（リスク追求）となる。

### 3.2 UIへの適用

| パターン | 実装例 | 効果 |
|---|---|---|
| 成功率フレーム | 「顧客満足度 95%」vs「不満足率 5%」 | ポジティブフレームが信頼感を向上させる |
| 節約フレーム | 「年払いで20%お得」vs「月払いだと20%余計にかかる」 | 利得フレームは安全策を、損失フレームは行動を促す |
| 時間フレーム | 「5分で完了」vs「たった300秒」 | 馴染みのある単位で短く感じさせる |
| 属性フレーム | 「脂肪分5%」vs「赤身95%」 | 同一商品でもポジティブ属性の強調で評価が向上する |
| 進捗フレーム | 「残り2ステップ」vs「6ステップ中4完了」 | 完了寄りのフレームが継続意欲を高める |

### 3.3 Apple HIG との接点

- **建設的メッセージング**: HIG はエラーメッセージにおいて「何ができないか」ではなく「何をすればよいか」を伝える肯定的フレーミングを原則としている
- **アラートの文言**: 破壊的アクションのアラートでも、ユーザーが取るべき行動を明確に示す前向きな表現が推奨されている

### 3.4 アンチパターン

- **恐怖訴求の乱用**: 「今すぐ対応しないとデータが消えます」のような過度な恐怖フレーミングは短期的に行動を促すが、長期的にはブランド信頼を毀損する
- **誤解を招くフレーミング**: 「最大90%OFF」と謳いながら対象商品がごく一部である場合、優良誤認に該当する
- **ネガティブフレームの常用**: 損失フレームは強力だが、多用するとユーザーに不安感やストレスを与え、離脱率が上昇する

### 3.5 実務活用

- コンバージョンに直結するCTAでは、ポジティブフレーム（「始める」「手に入れる」）を基本とする
- 解約防止画面では損失フレーム（「特典を失います」）が有効だが、脅迫的にならない範囲で使用する
- A/Bテストではフレーム違いのコピーを最低2パターン検証する
- ダッシュボードでは同一指標を「達成率」「未達率」両方で表示可能にし、文脈に応じて切り替えられる設計が望ましい

→ 関連: `choice.md` §4（損失回避）

## §4. 損失回避 + 保有効果（Loss Aversion / Endowment Effect）

> 人は同額の利得よりも損失を約2倍強く感じ、一度所有したものを手放すことに強い抵抗を示す。

### 4.1 心理学的メカニズム

**損失回避（Loss Aversion）**: Kahneman & Tversky（1979）のプロスペクト理論の核心。損失の心理的インパクトは同等の利得の約2〜2.5倍とされる。これは進化心理学的に「資源の喪失は生存に直結する」という適応メカニズムで説明される。

**保有効果（Endowment Effect）**: Thaler（1980）が命名。人は自分が所有するものに対して、市場価格よりも高い価値を付与する。実験ではマグカップの所有者は平均$7.12で売ろうとし、非所有者は平均$2.87で買おうとした（WTA-WTP格差）。

両者は密接に関連しており、保有効果は「手放すこと = 損失」と知覚されるために生じる。デジタルプロダクトにおいても、カスタマイズしたプロフィール、蓄積したデータ、獲得したステータスに対して保有効果が発生する。

### 4.2 UIへの適用

| パターン | 実装例 | 効果 |
|---|---|---|
| 無料トライアル | 30日間全機能を使わせた後、有料への切替を促す | 保有効果により解約率が低下する |
| 機能制限の可視化 | 「ダウングレードすると以下の機能が使えなくなります」と具体的にリスト表示 | 損失の具体化が解約を抑止する |
| ポイント残高表示 | 「3,240ポイント（¥3,240相当）が失効します」 | 保有ポイントの損失回避が購買を促進する |
| 進捗の可視化 | 「これまでの学習記録120時間分が引き継げなくなります」 | 蓄積投資の損失感が継続を動機づける |
| 破壊的アクション警告 | 削除確認ダイアログに「この操作は取り消せません」と明記 | 不可逆な損失の認知がミスを防止する |

### 4.3 Apple HIG との接点

- **破壊的アクションの確認**: HIG は削除・上書きなどの破壊的操作に対して、赤色ボタン + 確認ダイアログを必須としている。これは損失回避の心理を適切に活用したパターンである
- **Undo の提供**: HIG は可能な限り操作の取り消し（Undo）を提供し、損失リスクを軽減することで、ユーザーの探索行動を促進する設計を推奨している

### 4.4 アンチパターン

- **ダークアージェンシー**: 「カートの商品は5分後に削除されます！」のような虚偽または過度な緊急性表示はダークパターンであり、消費者保護法令に抵触する可能性がある
- **罪悪感の利用**: 「本当に退会しますか？あなたの仲間が寂しがります」のような感情的操作は、一時的に離脱を防いでも長期的なブランド毀損を招く
- **過度な損失フレーム**: 毎回「損失」を強調するUIは、ユーザーにストレスを与え、ネガティブなブランドイメージを形成する

### 4.5 実務活用

- トライアル設計では「使い始めの体験」を最大化し、保有効果を早期に醸成する
- 解約フローでは「失うもの」を具体的かつ事実に基づいて提示する（誇張しない）
- ポイントプログラムでは失効前の通知を段階的に送り、損失回避を活用した購買促進を行う
- データポータビリティ（エクスポート機能）を提供することで、ロックイン感を軽減しつつ信頼を構築する

→ 関連: `choice.md` §5（希少性効果）, `choice.md` §8（サンクコスト効果）

## §5. 希少性効果（Scarcity Effect）

> 手に入りにくいものほど価値が高いと感じる。限定性が欲求を増幅させる。

### 5.1 心理学的メカニズム

Cialdini（1984）の「影響力の武器」で体系化された6つの原則の一つ。希少性効果は2つの心理メカニズムで駆動される。

1. **商品理論（Commodity Theory）**: Brock（1968）の理論。利用可能性が制限された情報・商品は、その希少性に比例して価値が増大する
2. **心理的リアクタンス**: Brehm（1966）の理論。選択の自由が脅かされると、制限された選択肢への欲求が増大する。「手に入らなくなるかもしれない」という脅威が行動を加速させる

Worchel, Lee & Adewole（1975）のクッキー実験では、瓶に2枚のクッキーが残っている条件は10枚の条件よりも高い評価を受けた。さらに「以前は豊富だったが減少した」条件が最も高い評価を得た。

### 5.2 UIへの適用

| パターン | 実装例 | 効果 |
|---|---|---|
| 在庫数表示 | 「残り3点」を赤字で表示 | 購買決断の加速 |
| 期間限定 | 「本日23:59まで」のカウントダウンタイマー | 緊急性の醸成と即時行動の促進 |
| 限定アクセス | 「招待制・先着100名」の早期アクセスプログラム | 独占感による登録意欲の向上 |
| 閲覧者数表示 | 「現在15人がこの商品を見ています」 | 競争意識の喚起 |
| シーズン限定 | 「春季限定デザイン」「年末特別版」 | 時間的希少性による収集欲の刺激 |

### 5.3 Apple HIG との接点

- **控えめな緊急性表示**: HIG はバッジやインジケータを用いた情報提示を許容するが、過度にユーザーの注意を奪う表現（点滅・過剰なカウントダウン等）は避けるべきとしている
- **通知の節度**: 希少性を通知で伝える場合、HIG の通知ガイドラインに従い、ユーザーにとって真に有益な情報のみを送信する

### 5.4 アンチパターン

- **虚偽の希少性**: 実際には在庫が十分にあるのに「残りわずか」と表示する行為は、不当景品類及び不当表示防止法に違反する可能性がある
- **常態化した緊急性**: 365日「期間限定セール」を実施していると、ユーザーは「いつでも買える」と学習し、希少性効果が完全に失われる（「狼少年効果」）
- **過度な競争煽り**: 「他の人が今買おうとしています！」を常時表示すると、プレッシャーが不快感に転じてブランドイメージが低下する

### 5.5 実務活用

- 在庫数表示は事実に基づく正確な数値を使用する（法令遵守が大前提）
- カウントダウンタイマーは本当に締切が存在する場合にのみ使用する
- 希少性メッセージの表示閾値を定義する（例: 在庫5点以下で表示、50点以上では非表示）
- 「限定版」施策は年間カレンダーで計画し、乱発を避ける

→ 関連: `choice.md` §4（損失回避）, `motivation.md` §8（心理的リアクタンス — 過度な希少性は逆効果）

## §6. ナッジ効果（Nudge Effect）

> 選択の自由を奪わず、選択環境のデザインによって望ましい行動へ自然に導く。

### 6.1 心理学的メカニズム

Thaler & Sunstein（2008）の著書「Nudge」で提唱された概念。「リバタリアン・パターナリズム」の思想に基づき、「自由を制限せずに、より良い選択を促す」ことを目指す。

ナッジの理論的基盤はデュアルプロセス理論（Kahneman, 2011）にある。人間の思考はシステム1（自動的・直感的）とシステム2（意識的・分析的）に分かれ、日常の大半の判断はシステム1が担う。ナッジはシステム1に働きかけ、認知的努力を最小化しながら望ましい行動を促進する。

主要なナッジ手法:
- **デフォルト設定**: 最も推奨される選択肢を初期値にする
- **社会的規範の提示**: 「80%のユーザーがこの設定を選択しています」
- **フィードバックの即時化**: 行動の結果を即座に可視化する
- **選択肢の簡素化**: 複雑な意思決定を分解・整理する

### 6.2 UIへの適用

| パターン | 実装例 | 効果 |
|---|---|---|
| スマートデフォルト | 環境配慮オプション（ペーパーレス領収書）をデフォルトONに設定 | オプトアウト率は通常10%以下にとどまる |
| 社会的規範ナッジ | 「このエリアの85%の世帯がエコプランを選択」 | 社会的同調圧力が選択を誘導する |
| 視覚的ナッジ | 推奨プランを大きく表示し、「おすすめ」バッジを付与 | 視線誘導と権威性で選択率が向上する |
| 順序ナッジ | 健康的なメニューを一覧の上部に配置 | 系列位置効果と合わせて選択率が上昇する |
| フィードバックナッジ | エネルギー消費のリアルタイム表示 | 行動の結果を即時認知させ、節約行動を促進する |

### 6.3 Apple HIG との接点

- **ガイド付き設定**: iOS の設定画面は推奨設定を明示し、ユーザーが最適な構成に辿り着きやすいナッジ設計を採用している
- **インライン提案**: テキスト入力時の予測候補（QuickType）は、ユーザーの入力負担を軽減するナッジの典型例である
- **Screen Time**: 使用時間の可視化と制限の提案は、デジタルウェルビーイングへのナッジとして設計されている

### 6.4 アンチパターン

- **企業利益へのナッジ偽装**: ユーザーの利益ではなく企業の利益を最大化するためにナッジを悪用すること（例: メルマガ登録をデフォルトON、解除方法を分かりにくくする）はダークパターンに該当する
- **選択の自由の実質的剥奪**: オプトアウトの手順が極端に複雑（10ステップの解除手順等）な場合、形式上の自由は保たれていても実質的にナッジではなく強制である
- **透明性の欠如**: ナッジの存在をユーザーに隠蔽し、操作されていることに気づかせない設計は倫理的に問題がある

### 6.5 実務活用

- デフォルト値の設定は「ユーザーの最善の利益」を基準に決定する
- 全てのデフォルトに対して「変更する自由」が明確かつ簡単に行使できることを確認する
- ナッジの効果をA/Bテストで計測し、ユーザー満足度（NPS等）への影響も併せて追跡する
- GDPR・個人情報保護法の文脈では、プライバシー設定のデフォルトは「最もプライバシーを保護する」選択肢にすべきとされている

→ 関連: `information.md` §7（デフォルトバイアス）, `choice.md` §9（意図的フリクション — ナッジと対をなす概念）

## §7. 決断疲れ（Decision Fatigue）

> 意思決定を繰り返すほど判断の質が低下し、最終的には選択そのものを回避する。

### 7.1 心理学的メカニズム

Baumeister et al.（1998）の「自我消耗（ego depletion）」理論に基づく。意志力は有限のリソースであり、意思決定のたびに消耗する。消耗が進むと以下の3段階で劣化が生じる。

1. **判断の質の低下**: トレードオフの分析が浅くなり、単純な基準（価格のみ等）に依存する
2. **衝動的選択**: 短期的な快楽を優先し、長期的利益を軽視する
3. **選択回避（Choice Avoidance）**: Iyengar & Lepper（2000）の「ジャム研究」で実証。24種類のジャムを提示した場合の購買率は3%、6種類では30%。選択肢が多すぎると人は「選ばない」という選択をする

日常生活では1日に約35,000件の意思決定を行うとされ（Sahakian & Labuzetta, 2013）、重要な決定を1日の後半に行う場合は注意が必要である。

### 7.2 UIへの適用

| パターン | 実装例 | 効果 |
|---|---|---|
| 選択肢の制限 | 1画面に表示する選択肢を5±2個に制限する | ミラーの法則と合わせて認知負荷を抑制する |
| プログレッシブフィルタリング | カテゴリ → サブカテゴリ → 商品の段階的絞り込み | 一度の決定に必要な比較対象を削減する |
| スマートレコメンデーション | 「あなたへのおすすめ」として3〜5件を厳選提示 | 全商品から選ぶ負担を解消する |
| 保存して後で決める | 「お気に入り」「比較リスト」への保存機能 | 即時決定のプレッシャーを軽減する |
| クイックアクション | 「前回と同じ注文」「ワンクリック購入」 | 反復的な決定を自動化する |

### 7.3 Apple HIG との接点

- **キュレーションの原則**: App Store の「Today」タブは全アプリの一覧ではなく、編集部が厳選したコンテンツを提示し、選択肢過多を防止している
- **設定の簡素化**: iOS の設定項目は階層構造で整理され、一画面に表示される選択肢数を最小限に抑えている
- **Siri Suggestions**: 使用パターンに基づく提案により、ユーザーの意思決定コストを低減している

### 7.4 アンチパターン

- **過度な単純化**: パワーユーザーに必要なオプションまで隠蔽すると、カスタマイズ性の喪失に対する不満が生じる。段階的開示（progressive disclosure）で対応する
- **選択肢の恣意的制限**: ビジネス上の都合で本来必要な選択肢を削ると、ユーザーのニーズを満たせない
- **意思決定ポイントの連続配置**: ウィザード形式で10画面連続で選択を求めると、後半のステップで判断の質が著しく低下する

### 7.5 実務活用

- フォーム設計では1ステップあたりの意思決定を3つ以下に制限する
- 重要度の高い決定（プラン選択・支払い方法等）はフロー序盤に配置する
- リピートユーザーには前回の選択を記憶してデフォルトに反映する
- 管理画面設計では「よく使う操作」と「詳細設定」を分離し、段階的に提示する

→ 関連: `information.md` §1（認知負荷）, `information.md` §2（段階的開示）

## §8. サンクコスト効果（Sunk Cost Effect）

> 過去に投じた時間・金銭・労力が「もったいない」という感情を生み、非合理な継続を促す。

### 8.1 心理学的メカニズム

Arkes & Blumer（1985）が体系的に研究。経済学的には「埋没費用（sunk cost）は将来の意思決定に影響すべきではない」が合理的判断だが、人間は過去の投資を考慮して判断する。

このバイアスは複数の心理メカニズムで説明される。

1. **損失回避との連携**: 過去の投資を「損失」として認識することへの回避
2. **自己正当化欲求**: 過去の判断が「正しかった」と証明したいという動機
3. **完了欲求**: ツァイガルニク効果と重なり、未完了のプロジェクトへの心理的緊張が継続を促す

Staw（1976）の「エスカレーション・オブ・コミットメント」研究では、失敗プロジェクトへの追加投資が、責任者ほど大きくなることが示されている。

### 8.2 UIへの適用

| パターン | 実装例 | 効果 |
|---|---|---|
| 進捗バーの表示 | 「プロフィール完成度 70%」と残り項目を表示 | 既存の投資を可視化し完了動機を強化する |
| 累積実績の表示 | 「これまでの学習時間: 48時間」「走行距離: 320km」 | 蓄積の価値を認知させ継続を促進する |
| ロイヤルティポイント | 「あと2,000ポイントでゴールド会員」 | 既獲得ポイントのサンクコスト + 目標勾配が二重に作用する |
| ストリーク表示 | 「連続ログイン: 45日」 | ストリーク断絶への損失回避が毎日の行動を促す |
| コレクション進捗 | 「12/20 コンプリートまであと8つ」 | 収集済みアイテムのサンクコストが残りの収集を動機づける |

### 8.3 Apple HIG との接点

- **プログレスバー**: HIG はタスクの進捗を視覚的にフィードバックするプログレスインジケータを推奨しており、これはサンクコスト効果を正の方向で活用する好例である
- **アクティビティリング（Apple Watch）**: 毎日のムーブ・エクササイズ・スタンドリングの蓄積表示は、サンクコスト効果とストリーク心理を組み合わせたデザインである
- **実績バッジ**: Apple Fitness+ のバッジシステムは、達成済みバッジのコレクションがさらなる運動の動機となる設計

### 8.4 アンチパターン

- **解約トラップ**: サンクコストを悪用してユーザーを不利なサブスクリプションに閉じ込める設計（例: 解約するとこれまでのデータが全て削除される仕様）はダークパターンである
- **撤退コストの人為的増大**: プラットフォーム移行時にデータエクスポートを困難にし、サンクコストによるロックインを強化する手法は、デジタル市場法（DMA）等で規制が進んでいる
- **ストリーク脅迫**: 「ストリークが途切れます！」を過度に通知すると、義務感がストレスに転じ、アプリ自体を削除する動機になる

### 8.5 実務活用

- 進捗可視化は「ユーザーの達成感」を目的とし、離脱を防ぐための脅しとして使わない
- データポータビリティを確保し、「このサービスに閉じ込められている」という感覚を与えない
- ストリーク機能には「お休み券」や「復活機能」を組み込み、完璧主義のプレッシャーを緩和する
- 長期利用者向けの「振り返り」機能（年間レポート等）でサンクコストを肯定的に活用する

→ 関連: `choice.md` §4（損失回避）, `motivation.md` §4（ツァイガルニク効果）

## §9. 意図的フリクション（Intentional Friction）

> 敢えて操作に「摩擦」を加えることで、衝動的・有害な行動を防ぎ、慎重な判断を促す。

### 9.1 心理学的メカニズム

Cox et al.（2014）の研究に基づく設計原則。UXデザインでは一般に摩擦（friction）を排除することが良しとされるが、特定の文脈では意図的に摩擦を挿入することが、ユーザーの利益を保護する。

理論的基盤はデュアルプロセス理論にある。フリクションはシステム1（自動的・直感的）の即時反応を中断し、システム2（分析的・熟慮的）を起動させるスイッチとして機能する。これにより以下が実現される。

1. **衝動的行動の抑制**: 一呼吸置くことで感情的な判断を再考する機会が生まれる
2. **エラー防止**: 取り返しのつかない操作の前に確認を挟むことで、誤操作のリスクを低減する
3. **規範的行動の促進**: 利用規約の確認、年齢確認、同意取得において、形骸化を防ぐ

ナッジ効果（§6）が「望ましい行動を容易にする」アプローチであるのに対し、意図的フリクションは「望ましくない行動を困難にする」という対照的なアプローチをとる。両者は相補的に機能する。

### 9.2 UIへの適用

| パターン | 実装例 | 効果 |
|---|---|---|
| 入力確認 | アカウント削除時に「DELETE」とタイプさせる | 自動的な操作を強制的に中断し、意識的な確認を促す |
| 時間遅延 | 「送信取り消し」のための5秒カウントダウン（Gmailの送信取り消し） | 衝動的送信の直後に再考の時間を提供する |
| 二段階確認 | 「本当に削除しますか？」→「削除された場合、復元できません。続行しますか？」 | 重大さを段階的に認知させる |
| スワイプ確認 | iOSの「スライドして電源オフ」 | 偶発的な操作を防止する物理的フリクション |
| 冷却期間 | 「この注文は24時間以内であればキャンセルできます」 | 購入直後の後悔に対するセーフティネット |
| 情報提示フリクション | 送金前に「送金先: ○○ / 金額: ¥XX,XXX」の最終確認画面 | 認知的フリクションにより金額・宛先の誤りを検出する |

### 9.3 Apple HIG との接点

- **破壊的アクションのデザイン**: HIG は赤色のボタン（destructive style）、確認アラート、スワイプ操作による削除確認を体系的に定義しており、意図的フリクションの模範的実装である
- **Face ID / Touch ID 確認**: 生体認証は最も自然な形のフリクションであり、セキュリティと利便性を両立させている
- **App Tracking Transparency**: iOS 14.5 以降のトラッキング許可ダイアログは、プライバシー保護のために挿入された制度的フリクションの例である

### 9.4 アンチパターン

- **日常操作への過剰フリクション**: 毎回の保存操作に確認ダイアログを表示すると、ユーザーは確認を読まずにOKをクリックする習慣（habituation）が形成され、本当に重要な確認も無視されるようになる
- **理由のないフリクション**: なぜ追加の手順が必要なのかをユーザーに説明しないフリクションは、単なるUX劣化と知覚され、心理的リアクタンスを引き起こす
- **解約フリクションの悪用**: 解約手続きを意図的に複雑にする「ダークパターン」は、FTC（米国連邦取引委員会）が2023年に「click-to-cancel」規則で規制対象とした。サブスクリプションの解約は加入と同等の容易さであるべきとされている
- **フリクションの一貫性欠如**: 同じアプリ内で類似の操作に対するフリクションレベルが異なると、ユーザーのメンタルモデルが混乱する

### 9.5 実務活用

- フリクションを設計する際は「保護する対象」を明確にする（データの保護？ユーザーの金銭？プライバシー？）
- フリクションの段階を操作の重大度に比例させる: 低（確認ダイアログ）→ 中（入力確認）→ 高（時間遅延 + 入力確認）
- 頻度の高い操作にはフリクションを設けない。頻度が低く影響が大きい操作にフリクションを集中させる
- フリクション導入後はタスク完了率・エラー率・ユーザー満足度を計測し、過度でないか検証する
- 「元に戻せる操作」にはフリクションよりもUndo機能を提供するほうが効果的な場合が多い

→ 関連: `motivation.md` §8（心理的リアクタンス — フリクションが過度だと反発を招く）, `information.md` §1（認知負荷）

## 倫理ガイドライン — 選択設計における境界線

選択設計の各法則は強力な影響力を持つため、以下の倫理基準を遵守する。

| 原則 | 説明 |
|---|---|
| **透明性** | ユーザーに対して情報を隠蔽・歪曲しない。フレーミングは「事実の範囲内」で行う |
| **可逆性** | ユーザーが選択を変更・撤回できる手段を常に提供する |
| **比例性** | フリクションや希少性メッセージは、保護する利益の重大さに比例させる |
| **ユーザー利益の優先** | ナッジやデフォルトは「ユーザーの最善の利益」を基準に設計する |
| **法令遵守** | 景品表示法・特定商取引法・個人情報保護法・GDPR・DMA等の法令を遵守する |
