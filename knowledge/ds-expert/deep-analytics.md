## §1. 高度な時系列分析

### 1.1 状態空間モデル

**一般形**:
```
観測方程式: yₜ = Zₜαₜ + εₜ     (εₜ ~ N(0, Hₜ))
遷移方程式: αₜ₊₁ = Tₜαₜ + Rₜηₜ  (ηₜ ~ N(0, Qₜ))
```
- `αₜ`: 状態ベクトル（直接観測不能）
- `yₜ`: 観測ベクトル

**カルマンフィルター**:
1. **予測ステップ**: 前時点の情報から現時点の状態を予測
   ```
   αₜ|ₜ₋₁ = Tₜαₜ₋₁|ₜ₋₁
   Pₜ|ₜ₋₁ = TₜPₜ₋₁|ₜ₋₁Tₜ' + RₜQₜRₜ'
   ```
2. **更新ステップ**: 観測値で状態推定を修正
   ```
   Kₜ = Pₜ|ₜ₋₁Zₜ'(ZₜPₜ|ₜ₋₁Zₜ' + Hₜ)⁻¹  (カルマンゲイン)
   αₜ|ₜ = αₜ|ₜ₋₁ + Kₜ(yₜ - Zₜαₜ|ₜ₋₁)
   Pₜ|ₜ = (I - KₜZₜ)Pₜ|ₜ₋₁
   ```

**カルマンスムーザー**: 全時点のデータを使って過去の状態を再推定（事後的分析向け）。

**ARIMAとの関係**: ARIMA(p,d,q) は状態空間モデルの特殊ケースとして表現可能。状態空間モデルの方が:
- 欠損値の自然な処理
- 複数の構成要素（トレンド+季節+回帰）の同時推定
- 時変パラメータの表現

**拡張カルマンフィルター (EKF)**: 非線形の遷移/観測方程式に対応（1次テイラー展開で線形近似）。

**Unscented カルマンフィルター (UKF)**: シグマポイントを用いた非線形推定（EKFより精度高い）。

### 1.2 変化点検出

**問題**: 時系列のパラメータ（平均、分散、傾き等）が突然変化する時点を検出。

**PELT (Pruned Exact Linear Time)**:
- 動的計画法ベースの最適分割
- 枝刈りにより計算量O(n)を達成
- ペナルティ: BIC/AIC/MBIC

**Bayesian Online Changepoint Detection (BOCPD)**:
- 各時点で「前回の変化点からの経過時間（run length）」の事後分布を計算
- オンライン処理可能（逐次的にデータを取り込む）
- Hazard function: 変化点の事前発生率

```python
import ruptures
algo = ruptures.Pelt(model="rbf", min_size=5).fit(signal)
result = algo.predict(pen=10)
```

**CUSUM (Cumulative Sum)**: 制御図ベースの逐次検出法。製造業の品質管理で標準的。

### 1.3 動的因子モデル

**目的**: 多数の時系列から共通因子を抽出（次元削減 + 時間構造）。

```
yₜ = Λfₜ + εₜ      (観測方程式)
fₜ = Φfₜ₋₁ + ηₜ    (因子の時間発展)
```

- 静的因子分析の時系列版
- マクロ経済の景気指標抽出に広く使用
- EM アルゴリズム + カルマンフィルターで推定

### 1.4 高次元時系列

**VAR (Vector AutoRegression)**:
```
yₜ = c + A₁yₜ₋₁ + A₂yₜ₋₂ + ... + Aₚyₜ₋ₚ + εₜ
```
- 次数選択: AIC/BIC（低次元の場合）
- 高次元: LASSO-VAR、Elastic Net-VAR で正則化

**グレンジャー因果性**:
- 「xの過去の値がyの予測に有用か」を検定
- **因果性ではなく予測有用性**（名称に反して）
- ペアワイズ検定 + 多重比較補正、またはVAR全体でのF検定

**共和分 (Cointegration)**:
- 個々には非定常だが、線形結合が定常になる関係
- Johansen検定: 共和分ランクの決定
- VECM (Vector Error Correction Model): 長期均衡関係を組み込んだVAR

**インパルス応答関数 (IRF)**:
- あるショックが各変数に与える動的影響を可視化
- 直交化IRF: コレスキー分解（変数の順序に依存 → 注意）

### 1.5 実務活用: リアルタイム異常検知

- **IoTセンサー**: カルマンフィルターで予測値と実測値の乖離をモニタリング
- **金融**: BOCPD でレジームシフト（ボラティリティ変化）をリアルタイム検出
- **EC**: 動的因子モデルで売上・アクセス・在庫の共通トレンドを抽出
- **サプライチェーン**: VAR でリードタイム・需要・在庫の相互影響を分析

## §2. 空間統計

### 2.1 地理空間データの種類

| 種類 | 特徴 | 分析手法 |
|------|------|---------|
| 地理統計データ (Geostatistical) | 連続空間上の測定値 | クリギング、バリオグラム |
| 格子データ (Lattice) | 離散的な領域に集計 | 空間自己回帰モデル |
| 点過程データ (Point Pattern) | イベントの位置そのもの | K関数、強度推定 |

### 2.2 クリギング（空間補間）

**バリオグラム**: 距離に応じたデータの変動を表す。
```
γ(h) = (1/2N(h)) Σ [Z(sᵢ) - Z(sⱼ)]²
```
- **ナゲット (nugget)**: 距離0での不連続（測定誤差）
- **シル (sill)**: バリオグラムの上限値（全分散）
- **レンジ (range)**: 空間相関が消失する距離

**バリオグラムモデル**:
- 球面モデル（最も一般的）
- 指数モデル
- ガウスモデル（滑らかな空間場向き）

**クリギングの種類**:
| 種類 | 特徴 |
|------|------|
| Ordinary Kriging | 平均未知、局所的に推定 |
| Universal Kriging | 平均がトレンドを持つ |
| Co-Kriging | 複数変数の空間相関を同時利用 |

**予測**: BLUP (Best Linear Unbiased Predictor) — 不偏かつ分散最小の線形予測。

### 2.3 空間自己相関

**モランの I (Moran's I)**:
```
I = (n / Σᵢ Σⱼ wᵢⱼ) × (Σᵢ Σⱼ wᵢⱼ(xᵢ - x̄)(xⱼ - x̄)) / (Σᵢ(xᵢ - x̄)²)
```
- I > 0: 正の空間自己相関（類似値が近接）
- I < 0: 負の空間自己相関
- I ≈ 0: ランダム配置

**局所的指標 (LISA)**:
- 各地点での局所的なモランのI
- ホットスポット/コールドスポットの検出
- モランの散布図: (自身の値, 近傍の平均値) の散布図 → 4象限に分類

**空間的加重行列 W**:
- 距離ベース: `wᵢⱼ = 1/dᵢⱼ²` （距離減衰）
- 隣接ベース: `wᵢⱼ = 1` if 隣接, 0 otherwise（Queen/Rook隣接）
- k近傍: 最近傍k個のみ非ゼロ

### 2.4 空間回帰モデル

**SAR (Spatial Autoregressive Model)**:
```
y = ρWy + Xβ + ε
```
- 空間ラグモデル: 近傍のyが自身のyに影響

**SEM (Spatial Error Model)**:
```
y = Xβ + u,  u = λWu + ε
```
- 空間的に相関した誤差（残差の空間パターン）

**モデル選択**: LM検定（ラグ vs エラー）。Robust版も使用。

### 2.5 地理的加重回帰 (GWR)

**特徴**: 回帰係数が空間的に変化（非定常性のモデリング）。

```
yᵢ = β₀(uᵢ,vᵢ) + β₁(uᵢ,vᵢ)x₁ᵢ + ... + εᵢ
```

- `(uᵢ, vᵢ)`: 地点の座標
- 各地点で局所的な重み付き回帰を実施
- **バンド幅選択**: CV（交差検証）またはAICc

**多重共線性**: GWRでは局所的なサンプルサイズが小さいため、多重共線性が深刻化しやすい。VIF（分散膨張係数）の確認が必須。

### 2.6 実務活用: 立地分析、環境モニタリング

- **店舗立地**: GWRで地域ごとの需要要因の効果を推定（人口の効果が都市と郊外で異なる等）
- **不動産価格**: クリギングで未測定地点の地価を補間
- **大気汚染**: Co-Krigingで気象データとPM2.5の空間相関を利用した高精度推定
- **感染症**: LISA でクラスター（ホットスポット）検出 → 重点介入地域の特定
- **農業**: バリオグラム分析で土壌特性の空間変動パターン把握

## §3. ネットワーク分析

### 3.1 グラフ理論の基礎

**グラフの種類**:
| 種類 | 特徴 | 例 |
|------|------|-----|
| 無向グラフ | 辺に方向なし | 友人関係 |
| 有向グラフ | 辺に方向あり | フォロー関係、引用 |
| 重み付きグラフ | 辺に重み | 取引額、通信頻度 |
| 二部グラフ | 2種のノード間の辺のみ | ユーザ-商品 |

**基本統計量**:
- **次数分布**: スケールフリー（べき乗則）→ ハブノードの存在
- **クラスタ係数**: 友人の友人も友人である確率
- **最短経路長**: スモールワールド性（平均パス長が短い）
- **直径**: グラフ内の最長最短経路

### 3.2 中心性指標

| 中心性 | 定義 | 解釈 |
|--------|------|------|
| 次数中心性 | 接続数 / (N-1) | 直接のつながりの多さ |
| 媒介中心性 | 最短経路上に位置する割合 | 情報のブリッジ・仲介者 |
| 近接中心性 | 他ノードへの平均最短距離の逆数 | 情報到達の速さ |
| 固有ベクトル中心性 | 重要なノードとつながるほど高い | 影響力の波及 |
| PageRank | 重要ページからリンクされるほど高い | Web上の権威性 |
| Katz中心性 | 全パス（減衰付き）の合計 | 長距離の影響も考慮 |

**使い分けの判断基準**:
- 直接の人脈が重要 → 次数中心性
- 組織内のゲートキーパー発見 → 媒介中心性
- 情報拡散の起点 → 近接中心性
- Web/SNSの影響力 → PageRank/固有ベクトル中心性

### 3.3 コミュニティ検出

**モジュラリティ (Modularity) Q**:
```
Q = (1/2m) Σᵢⱼ [Aᵢⱼ - kᵢkⱼ/(2m)] δ(cᵢ, cⱼ)
```
- Q > 0.3: 有意なコミュニティ構造
- NP困難問題 → 近似アルゴリズムが必要

**ローバン法 (Louvain)**:
1. 各ノードを個別のコミュニティに
2. 隣接コミュニティへの移動でQ最大の操作を貪欲に実行
3. コミュニティをノードに集約
4. 2-3を繰り返し

**Leiden法**: ローバンの改良版。接続性保証 + 最適化の質向上。

**スペクトラルクラスタリング**: ラプラシアン行列の固有ベクトルで埋め込み→k-means。

**重複コミュニティ**: 1ノードが複数コミュニティに所属可能（BigCLAM等）。

### 3.4 PageRank

```
PR(v) = (1-d)/N + d Σ_{u→v} PR(u)/L(u)
```
- `d`: ダンピング係数（通常0.85）
- `L(u)`: ノードuの出次数
- 反復計算で収束させる（固有値問題としても解ける）

**Personalized PageRank**: 特定ノードからのランダムウォークで重要度計算。推薦システムに応用。

### 3.5 GNN (Graph Neural Network)

**Message Passing Framework**:
```
h_v^(k) = UPDATE(h_v^(k-1), AGGREGATE({h_u^(k-1) : u ∈ N(v)}))
```

**主要アーキテクチャ**:
| モデル | 集約方法 | 特徴 |
|--------|---------|------|
| GCN | 平均 + 正規化 | シンプル、半教師あり学習 |
| GAT | Attention重み付き | ノードごとに重要度を学習 |
| GraphSAGE | サンプリング + 集約 | 大規模グラフ向け、帰納的学習 |
| GIN | 和 + MLP | 表現力がWLテストと同等 |

**グラフレベルタスク**:
- ノード分類: 各ノードのラベル予測
- リンク予測: 未観測の辺を予測
- グラフ分類: グラフ全体のラベル予測（ReadoutでグラフレベルのEmbedding）

**Over-smoothing問題**: 層を深くすると全ノードの表現が似てしまう。対策: 残差結合、JK-Net、DropEdge。

### 3.6 実務活用: SNS分析、不正検知

- **SNS影響力分析**: PageRank/固有ベクトル中心性でインフルエンサー特定
- **不正取引検知**: GNN で取引ネットワーク上のノード分類（正常/不正）
- **組織分析**: 媒介中心性で情報のボトルネック発見→組織構造改善
- **推薦**: 二部グラフ上のリンク予測（ユーザ→未購入商品の推薦）
- **サプライチェーン**: コミュニティ検出でサプライヤーグループの特定、リスク伝播分析

## §4. 自然言語処理（高度）

### 4.1 Transformer詳細

**入力表現**:
```
Input = Token Embedding + Position Embedding + (Segment Embedding)
```

**Multi-Head Attention**:
```
MultiHead(Q,K,V) = Concat(head₁,...,headₕ)Wᴼ
headᵢ = softmax(QᵢKᵢᵀ/√dₖ)Vᵢ
```
- ヘッド数h: 各ヘッドが異なる関係パターンを学習
- dₖ = d_model / h（ヘッドあたりの次元数）

**Layer Normalization**: バッチ正規化と異なり、特徴次元に沿って正規化（バッチサイズに依存しない）。

**Feed-Forward Network**: `FFN(x) = max(0, xW₁+b₁)W₂+b₂` （各位置独立に適用）。

**Encoder vs Decoder**:
| 構造 | マスク | 用途 |
|------|--------|------|
| Encoder | なし（双方向） | BERT型（理解系タスク） |
| Decoder | Causal mask（左方向のみ） | GPT型（生成系タスク） |
| Encoder-Decoder | Cross-Attention | 翻訳、要約 |

### 4.2 BERT/GPTファミリーの違い

| 特性 | BERT | GPT |
|------|------|-----|
| 構造 | Encoder | Decoder |
| 事前学習 | MLM + NSP | 自己回帰言語モデル |
| 文脈 | 双方向 | 左→右のみ |
| 得意タスク | 分類、固有表現抽出、QA | テキスト生成、対話 |
| ファインチューニング | タスク固有ヘッドを追加 | プロンプトエンジニアリング/Few-shot |

**BERTの派生**:
- RoBERTa: 動的マスキング、NSP除去、大バッチ
- ALBERT: パラメータ共有で軽量化
- DeBERTa: Disentangled Attention（内容と位置を分離）
- DistilBERT: 知識蒸留で50%軽量化

### 4.3 ファインチューニング戦略

**全層ファインチューニング**: 全パラメータを更新。小データでは過学習リスク。

**LoRA (Low-Rank Adaptation)**:
```
W' = W + BA  (B: d×r, A: r×k, r << min(d,k))
```
- 元の重み W は凍結
- 低ランク行列 BA のみ学習（パラメータ数: 2×d×r）
- VRAM節約 + 過学習抑制

**アダプタ (Adapter)**:
- Transformer層間に小さなMLPモジュールを挿入
- 元のパラメータは凍結

**Prefix Tuning**: 各層のKey/Valueに学習可能なプレフィックスベクトルを追加。

**手法選択**:
| 条件 | 推奨手法 |
|------|---------|
| 大規模データ + 十分なGPU | 全層ファインチューニング |
| 中規模データ + GPU制限 | LoRA (r=8-64) |
| 超大規模モデル | QLoRA (量子化+LoRA) |
| APIのみアクセス | プロンプトエンジニアリング |

### 4.4 RAG (Retrieval Augmented Generation)

**アーキテクチャ**:
1. **Indexing**: ドキュメントをチャンク分割 → Embedding → ベクトルDB保存
2. **Retrieval**: クエリをEmbedding → ベクトル検索で関連チャンク取得
3. **Generation**: 取得チャンク + クエリを LLM に入力 → 回答生成

**チャンク戦略**:
- 固定長: シンプルだが意味境界を無視
- セマンティック: 意味のまとまりで分割（再帰的分割）
- オーバーラップ: 隣接チャンクと重複させて文脈維持

**Embedding モデル**: sentence-transformers（SBERT系）、E5、BGE。

**ベクトルDB**: FAISS（ローカル）、Pinecone/Weaviate/Qdrant（クラウド）。

**Re-ranking**: 初期検索結果をCross-Encoderで再スコアリング（精度向上）。

**評価**: Faithfulness（回答の根拠性）、Relevance（検索結果の関連性）、RAGAS フレームワーク。

### 4.5 評価指標

| 指標 | 対象タスク | 計算 |
|------|-----------|------|
| BLEU | 翻訳 | n-gramの適合率（修正） |
| ROUGE-L | 要約 | 最長共通部分列ベースの再現率 |
| BERTScore | 汎用 | BERTのEmbeddingのコサイン類似度 |
| Perplexity | 言語モデル | exp(-1/N Σ log P(wₜ)) |

**BLEU の注意点**: 流暢性は測るが、意味的正確性は不十分。人間の評価との相関は限定的。

### 4.6 実務活用: 文書分類、質問応答、要約

- **法務**: BERT ファインチューニングで契約書のリスク条項自動分類
- **カスタマーサポート**: RAG で社内マニュアルを参照した自動回答
- **メディア**: GPT系モデルで記事要約の自動生成
- **採用**: 職務経歴書の自動スクリーニング（固有表現抽出 + 分類）
- **コンプライアンス**: メール/チャットの自動監視（感情分析 + キーワード検出）

## §5. 画像解析（高度）

### 5.1 物体検出

**2段階検出器**:
- **Faster R-CNN**: RPN(Region Proposal Network) → ROI Pooling → 分類/回帰
  - アンカーボックス: 複数のスケール/アスペクト比
  - 精度が高いが推論速度がやや遅い

**1段階検出器**:
- **YOLO (You Only Look Once)**: グリッド分割 → 各セルで同時に検出
  - YOLOv8/YOLO11: 最新版。精度と速度のバランスに優れる
  - リアルタイム処理向き
- **SSD**: マルチスケールの特徴マップから検出

**評価指標**:
- **mAP (mean Average Precision)**: 各クラスのAP（Precision-Recall曲線下面積）の平均
- **IoU (Intersection over Union)**: 予測ボックスと正解ボックスの重なり率
- **mAP@0.5**: IoU > 0.5 で検出成功とみなす場合のmAP
- **mAP@[0.5:0.95]**: IoU閾値を0.05刻みで変化させた平均（COCO標準）

**NMS (Non-Maximum Suppression)**: 重複する検出ボックスを除去。Soft-NMSは重複をスコア減衰で処理。

### 5.2 セグメンテーション

| 種類 | 出力 | 例 |
|------|------|-----|
| セマンティック | ピクセル単位のクラス | 各ピクセルが「道路」「建物」等 |
| インスタンス | 個別物体の領域 | 「犬A」「犬B」を区別 |
| パノプティック | 上記の統合 | stuff + things |

**主要アーキテクチャ**:
- **U-Net**: エンコーダ-デコーダ + スキップ接続。医用画像で標準
- **DeepLab v3+**: Atrous Convolution + ASPP（マルチスケール）
- **Segment Anything (SAM)**: Foundation Model。プロンプトベースのゼロショットセグメンテーション

**損失関数**:
- Cross-Entropy（ピクセル単位）
- Dice Loss: `1 - 2|P∩G|/(|P|+|G|)` — クラス不均衡に強い
- Focal Loss: 簡単なサンプルの重みを下げる

### 5.3 画像生成

**Diffusion Model**:
1. **Forward process**: データにガウスノイズを段階的に追加
2. **Reverse process**: ニューラルネットでノイズを段階的に除去
3. **Sampling**: 純粋なノイズから段階的にデノイズして画像生成

**主要モデル**:
- **DDPM**: 基本的なDiffusion
- **Stable Diffusion**: 潜在空間でDiffusion（計算効率向上）+ CLIP条件付け
- **DALL-E**: テキスト→画像生成

**GANとの比較**:
| 特性 | GAN | Diffusion |
|------|-----|-----------|
| 学習安定性 | 不安定 | 安定 |
| 多様性 | モード崩壊リスク | 高い |
| 生成速度 | 高速（1回のフォワードパス） | 遅い（反復デノイズ） |
| 画質 | 高い | 非常に高い |

### 5.4 実務活用: 製造業の外観検査

- **不良品検出**: YOLOで欠陥領域を検出（傷、汚れ、変形）
- **異常検知**: オートエンコーダで正常画像を学習→再構成誤差で異常判定
- **少数データ対応**: 転移学習（ImageNetモデル→ファインチューニング）+ データ拡張
- **半導体ウェーハ**: セマンティックセグメンテーションで欠陥種別の領域分類
- **医療画像**: U-Netで臓器セグメンテーション、Diffusion Modelでデータ拡張

## §6. 推薦システム

### 6.1 協調フィルタリング

**ユーザーベース**:
```
r̂ᵤᵢ = r̄ᵤ + Σ_{v∈N(u)} sim(u,v)(rᵥᵢ - r̄ᵥ) / Σ|sim(u,v)|
```
- sim: ピアソン相関 or コサイン類似度

**アイテムベース**:
```
r̂ᵤᵢ = Σ_{j∈rated(u)} sim(i,j)rᵤⱼ / Σ|sim(i,j)|
```

**比較**:
| 方式 | アイテム数 >> ユーザ数 | ユーザ数 >> アイテム数 |
|------|---------------------|---------------------|
| ユーザーベース | 計算コスト高 | 推奨 |
| アイテムベース | 推奨 | 計算コスト高 |

**コールドスタート問題**: 新ユーザ/新アイテムは履歴がなく推薦不可。

### 6.2 行列分解

**SVD (Singular Value Decomposition)**:
```
R ≈ UΣVᵀ  →  r̂ᵤᵢ = pᵤᵀqᵢ + μ + bᵤ + bᵢ
```
- pᵤ: ユーザの潜在因子ベクトル
- qᵢ: アイテムの潜在因子ベクトル
- μ, bᵤ, bᵢ: バイアス項

**NMF (Non-negative Matrix Factorization)**:
- 非負制約 → 解釈しやすい潜在因子
- トピックモデルとしても使用可能

**ALS (Alternating Least Squares)**:
- U固定→V更新、V固定→U更新を交互に
- 暗黙的フィードバック（クリック、購入）に対応（Implicit ALS）
- 大規模データの分散処理に適する

**損失関数**:
```
L = Σ_{(u,i)∈observed} (rᵤᵢ - pᵤᵀqᵢ)² + λ(||pᵤ||² + ||qᵢ||²)
```

### 6.3 コンテンツベース

**原理**: アイテムの特徴量（テキスト、カテゴリ、属性）とユーザプロファイルの一致度で推薦。

**手法**:
- TF-IDF + コサイン類似度（テキストベース）
- BERT/SBERTのEmbedding類似度
- ユーザプロファイル: 過去に高評価したアイテムの特徴平均

**利点**: コールドスタートに強い（アイテム属性があれば推薦可能）。
**欠点**: セレンディピティに乏しい（既知の嗜好に偏る）。

### 6.4 ハイブリッド手法

| 手法 | 説明 |
|------|------|
| 重み付きハイブリッド | CFとCBのスコアの加重平均 |
| スイッチング | 条件に応じてCF/CBを切り替え |
| カスケード | CBで候補絞り込み→CFでランキング |
| 特徴量結合 | CF特徴とCB特徴を結合して1つのモデルに |

**深層学習ベース**:
- **Neural CF (NCF)**: ユーザ×アイテムの相互作用をMLPで学習
- **Wide & Deep**: 記憶（Wide: 線形）+ 汎化（Deep: DNN）
- **Two-Tower**: ユーザタワー + アイテムタワー → 内積で効率的な推薦

### 6.5 評価指標

| 指標 | 説明 |
|------|------|
| Precision@K | 上位K推薦中の正解率 |
| Recall@K | 正解のうち上位K推薦に含まれる割合 |
| NDCG@K | 順位に割引をかけた適合度 |
| MAP | 各ユーザのAPの平均 |
| Hit Rate | 少なくとも1つ正解を含むユーザの割合 |
| 多様性 | 推薦リスト内のアイテム間距離 |
| カバレッジ | 全アイテム中推薦されたアイテムの割合 |

**オフライン vs オンライン評価**:
- オフライン: Hold-out / 時間分割CV
- オンライン: A/Bテスト（CTR、CVR、エンゲージメント）

### 6.6 実務活用: ECサイト、動画配信

- **EC**: アイテムベースCF（「この商品を買った人はこれも」）+ ハイブリッド
- **動画配信**: Two-Tower（候補生成）→ Neural CF（ランキング）の2段階
- **ニュース**: バンディット（探索-活用）でCTR最大化 + コンテンツベースの初期推薦
- **音楽**: NMFで潜在ジャンル抽出 + 協調フィルタリングでパーソナライズ
- **求人**: スキルEmbedding + 協調フィルタリングのハイブリッド
