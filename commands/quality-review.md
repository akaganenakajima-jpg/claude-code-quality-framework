# /quality-review — PDCA完了状況チェック

現在のタスクについて、PDCAの各段階の完了状況を確認する。

## チェックリスト

### Plan（計画）
- [ ] 受入基準が明文化されているか
- [ ] リスク判定を実施したか（🟢🟡🟠🔴）
- [ ] 影響範囲を特定したか

### Do（実行）
- [ ] テストを先に書いたか（TDD対象の場合）
- [ ] 仮説を立てたか（予測モデルの場合）
- [ ] 品質ゲートを通過したか
- [ ] 1機能/1バグ = 1コミットを守っているか

### Check（検証）
- [ ] ユニットテストが通るか
- [ ] E2E/統合テストが通るか（該当する場合）
- [ ] `git diff HEAD` で変更内容を確認したか
- [ ] 変更の要点をユーザーに報告したか

### Act（改善）
- [ ] 問題が発生した場合、根本原因を分析したか
- [ ] 是正処置を実施したか
- [ ] 水平展開（同じパターンの問題の確認）をしたか

## 実行方法

1. 上記チェックリストを現在のタスクに対して評価する
2. 未完了の項目があれば指摘する
3. 品質記録（`quality/metrics.md`）の更新が必要か確認する
