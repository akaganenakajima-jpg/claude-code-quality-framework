# /risk — リスクレベル自動判定

git diff を分析し、変更のリスクレベルを判定する。

## 手順

1. `git diff --stat` と `git diff --cached --stat` を実行して変更量を取得する
2. 変更対象ファイルの種類を分析する
3. 以下の基準でリスクレベルを判定する

## 判定基準

### 🟢 低リスク
- 新しいファイルの追加のみ
- テストの追加のみ
- コメント・ドキュメントの修正のみ
- 変更量: 20行以下 / 2ファイル以下

### 🟡 中リスク
- 既存コードの修正（機能は同じ）
- 設定ファイルの変更（元に戻せる範囲）
- 変更量: 100行以下 / 5ファイル以下
- **対応**: 先にコミットしてから着手

### 🟠 高リスク
- 複数の機能やAPIに影響する変更
- 外部連携（API・Webhook等）の変更
- 変更量: 300行以下 / 10ファイル以下
- **対応**: 影響範囲を洗い出してから着手

### 🔴 最高リスク
- データベース構造の変更
- cron/定期実行の変更
- 本番デプロイ
- 変更量: 300行超 / 10ファイル超
- **対応**: Worktree（隔離環境）で作業

## 出力

判定結果と推奨アクションを表示する。
詳細 → `quality/risks.md`
